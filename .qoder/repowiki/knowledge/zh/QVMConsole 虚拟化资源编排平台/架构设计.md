- **依赖倒置与注册模式**：采用 `Deps` 结构体（如 `vm/deps.go`, `clone/deps.go`）和全局 Hook 变量（如 `service/hooks.go`）打破 Go 包之间的循环引用。根包在启动时通过 `_register.go` 文件将核心逻辑注入子包，子包则通过接口或函数指针回调根包功能。
- **异步任务总线**：内置 `taskqueue` 模块作为中央调度器，统一管理克隆、迁移、快照等长耗时操作。`main.go` 负责集中注册所有业务处理器，确保后台任务与 HTTP 请求处理解耦。
- **启动态基础设施自愈**：在 `main.go` 启动序列中，系统会依次执行 OVS 网络修复、VPC 绑定恢复、端口转发规则重载及宿主机资源同步，确保服务重启后虚拟化环境的一致性。
- **分层路由与中间件链**：`router/router.go` 定义了基于角色的访问控制（RBAC），通过 `AdminMiddleware`、`ElasticCloudOnlyMiddleware` 等组合中间件实现细粒度的 API 权限隔离，并统一集成限流与 CORS 策略。

# VNC 控制台

通过 noVNC 在浏览器中远程访问虚拟机桌面。

## 架构

```
浏览器 (noVNC) ──WebSocket──▷ Go 后端 ──Unix Socket──▷ QEMU VNC
```

- VNC 使用 **Unix Socket** 模式，不暴露 VNC 端口到网络
- 后端通过 WebSocket 代理将 noVNC 流量转发到 VM 的 Unix Socket
- WebSocket 认证通过 URL query 参数传递 JWT Token

## 使用说明

### 1. 开启 VNC

进入虚拟机详情页 → 点击「VNC 控制台」标签页 → 点击「开启 VNC」按钮。

- 可选设置 VNC 密码（最长 8 位，留空则无密码）
- 开启 VNC 需要**重启虚拟机**才能生效
- VNC 会保留当前虚拟机的显示设备配置，不再强制改写显卡模型
- Windows 或 VMware 嵌套虚拟化环境中，如果 VNC 画面明显卡顿，可在虚拟机高级设置里优先尝试 `VGA` 或 `VMVGA`

### 2. 连接

VNC 开启后，点击「连接」按钮即可在浏览器中查看虚拟机桌面。

**工具栏功能：**
- **Ctrl+Alt+Del** - 向虚拟机发送组合键
- **常用组合键** - 支持快速发送 `Ctrl+Shift+Esc`、`Alt+Tab`、`Win+R`、`Ctrl+Alt+F1`、`Ctrl+Alt+F2`、`Ctrl+Alt+Backspace`
- **粘贴密码** - 如果当前虚拟机已保存登录密码，可一键将密码输入到 VNC 当前焦点位置
- **发送文本** - 手动输入任意文本后，按字符自动输入到 VNC 当前焦点位置，适合密码、命令、激活码和多行内容
- **全屏** - 全屏显示 VNC 画面
- **新窗口打开** - 在独立的浏览器窗口中打开 VNC 控制台，脱离面板侧边栏，自动连接并占满整个窗口
- **断开** - 断开 VNC 连接

### 2.1 粘贴密码说明

- “粘贴密码”使用的是虚拟机详情页中已保存的**来宾系统登录密码**
- 该功能会把密码发送到 **当前 VNC 焦点位置**，适合登录框、终端或应用密码输入框
- 这不是 VNC 自身认证密码；如果连接 VNC 时出现认证弹窗，仍需输入 VNC 密码
- 如果未保存虚拟机登录密码，按钮会自动禁用

### 2.2 发送文本说明

- “发送文本”支持在面板内嵌 VNC 和独立窗口 VNC 中使用
- 输入内容后，面板会按字符逐个发送到 **当前 VNC 焦点位置**
- 支持常见字母、数字、符号、空格、Tab 和换行；未显式映射的字符会按 Unicode 字符发送
- 适合粘贴命令、激活码、初始化脚本片段或需要多行输入的内容
- 如果目标系统对快捷键布局或输入法有特殊要求，建议先在目标输入框中确认焦点和输入法状态

### 3. 管理

通过右侧「管理」下拉菜单：
- **修改密码** - 即时生效，无需重启虚拟机
- **关闭 VNC** - 需重启虚拟机生效

## 性能建议

- VNC 更适合安装、救援、基础控制台操作，不适合作为 Windows 日常桌面远程协议
- Windows 虚拟机装好系统后，建议优先启用并使用 `RDP`
- 若宿主本身运行在 VMware 等上层虚拟化环境中，嵌套 KVM 的 VNC 延迟通常会比物理宿主更明显
- 如果必须继续使用 VNC，请优先调整虚拟机的显示设备模型，再观察是否改善

## 前端依赖

| 包名 | 用途 |
|------|------|
| `@novnc/novnc` | noVNC 客户端库，提供 RFB 协议连接 |

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/vm/:name/vnc/status` | 获取 VNC 状态 |
| POST | `/api/vm/:name/vnc/enable` | 开启 VNC |
| POST | `/api/vm/:name/vnc/disable` | 关闭 VNC |
| POST | `/api/vm/:name/vnc/passwd` | 修改 VNC 密码 |
| GET (WS) | `/api/vm/:name/vnc/ws` | noVNC WebSocket 代理 |

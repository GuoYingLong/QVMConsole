# 轻量云用户

## 功能概述

轻量云是普通用户下的新云类型，面向“管理员登记一台或多台云服务器配置，用户确认后开通使用”的场景。系统仍保留 `role=admin/user`，通过 `cloud_type` 区分：

| 类型 | 说明 |
|------|------|
| `elastic` | 弹性云，当前普通用户默认类型，继续使用用户级资源配额和自助创建能力 |
| `lightweight` | 轻量云，不走用户级配额，管理员按单台待开通 VM 设置网络、端口转发和快照上限 |

## 管理员流程

1. 在网络管理中先创建一个 NAT 类型 VPC，作为轻量云专用网络（仅注册新 VM 时需要）。
2. 在用户管理中创建或编辑普通用户，选择"轻量云"。
3. 选择 VM 来源：
   - **选择已有 VM**：从系统中已存在的 VM 列表中选择要分配的 VM，无需选择专用 VPC。
   - **注册新 VM**：需要先选择专用 VPC，然后点击"注册 VM"，从模板克隆表单登记待开通服务器配置。注册模式只允许模板克隆，可选择虚拟机硬盘存储位置，并自动使用该用户的专用 VPC，管理员不需要选择安全组。
4. 如果选择已有 VM，需要为每台 VM 设置：
   - 下行月流量配额，单位 GB，`0` 表示不限。
   - 上行月流量配额，单位 GB，`0` 表示不限。
   - 下行带宽，单位 Mbps。
   - 上行带宽，单位 Mbps。
   - 端口转发上限，`0` 表示不限。
   - 快照上限，默认 `2`，`0` 表示不限。
   - 运行时长配额，单位小时，`0` 表示不限。
5. 如果选择注册新 VM，在注册表单中为每台 VM 设置：
   - 下行月流量配额，单位 GB，`0` 表示不限。
   - 上行月流量配额，单位 GB，`0` 表示不限。
   - 下行带宽，单位 Mbps。
   - 上行带宽，单位 Mbps。
   - 端口转发上限，`0` 表示不限。
   - 快照上限，默认 `2`，`0` 表示不限。
   - 运行时长配额，单位小时，`0` 表示不限。
   - FnOS 模板可选择是否保留或指定设备 ID；保留或指定时用户确认开通后仍会注入登录凭据，但不会重置模板内的 `/etc/machine-id`，指定值会写入 `/usr/trim/etc/machine_id` 和 `/etc/device_id`。

用户管理的"注册VM"窗口会展示该用户所有轻量云 VM，包括待确认、开通中、失败和已开通记录。管理员可在这里编辑单台 VM 的流量、带宽、端口转发上限、快照上限和运行时长配额；已开通 VM 保存后会立即更新运行配额并重新应用带宽限制。已开通 VM 支持"移除"，该操作只移除注册列表记录、单 VM 配额和月流量状态，不删除虚拟机本体。

保存后配置进入待确认列表，并随邀请邮件或开通确认邮件发送给用户。邮件只列出配置清单，不包含 VM 登录密码。

用户登录面板后，在 VM 列表的“待开通服务器”卡片中逐台补全登录凭据并确认。确认后系统提交任务队列开通服务器，并自动写入 VM 归属、轻量云单 VM 配额、专用 VPC、专属安全组和登录凭据。

## 用户侧能力

轻量云用户只显示虚拟机列表。VM 列表不显示新建、编辑、删除、导入导出等入口，但会显示“待开通服务器”卡片。

待开通服务器确认规则：

- Linux/fnOS 模板：用户填写登录用户名和强密码。
- Windows 模板：用户名固定为 `administrator`，用户只填写强密码。
- 页面保留随机强密码按钮，密码不会通过邮件发送。
- 点击确认开通时会走现有高风险验证机制。

VM 详情页保留：

- 电源操作、VNC、快照、救援系统、重置密码、监控图表。
- 网络管理中的端口转发、专属安全组规则、单 VM 流量和带宽状态。

VM 详情页隐藏：

- VPC 切换、带宽编辑、静态 IP、定时任务、开发者页入口。

轻量云用户不能手动编辑静态 IP；首次开通端口转发时，后端会自动把当前 VM 正在使用的 VPC 地址固化为静态 IP，确保后续 DNAT 目标稳定。

## 配额规则

轻量云的流量配额、运行时长配额、端口转发配额和快照配额都按单台 VM 计算，不占用弹性云用户级配额。

流量统计使用 `vm_stats_records.net_rx_bytes/net_tx_bytes` 的本月相邻正增量，VM 重启导致计数器归零时不会产生异常增量。

VM 详情页中的轻量云实时下行/上行使用该 VM 的网卡统计值计算，不使用 VPC 交换机聚合流量。轻量云带宽限制同样按 VM 网卡口径生效：运行中的 VM 会在对应 `vnet` 接口上应用下行 egress HTB 限速，上行通过 IFB + HTB 做 ingress 整形，避免因 VPC 网关、旧 DHCP 租约或 OVS 流表未命中导致配置存在但实际不限速，同时减少 ingress police 丢包限速造成的单 TCP 流吞吐下降。轻量云运行态限速只使用 TC/IFB，`domiftune` 仅保存持久化配置，避免低速惩罚时 libvirt live burst 与 TC 队列叠加造成周期性卡顿。

当某台轻量云 VM 本月某方向流量超过配额：

- 下行超限只把下行限速到 `1Mbps`。
- 上行超限只把上行限速到 `1Mbps`。
- 未超限方向继续使用管理员设置的 VM 带宽。
- 管理员调大该方向配额且当前已用低于新配额时，会自动解除该方向限速。
- 月初自动恢复正常带宽。

惩罚速率默认是 `1Mbps`，可通过系统设置表的 `lightweight_traffic_penalty_mbps` 调整为更适合生产交互体验的速率。

## 单 VM 运行时长配额

轻量云新增按单台 VM 生效的运行时长配额：

- 管理员可为每台轻量云 VM 设置运行时长配额，单位为小时。
- `0` 表示不限。
- 计时状态包括 `running` 和 `paused`。
- 面板会把累计值持久化到 `lightweight_vm_quotas`，面板重启后不会清零。

达到配额后的行为：

- 该 VM 会自动提交异步关机任务。
- 后续再次开机会被拒绝，直到管理员上调该 VM 的运行时长配额。
- 自动关机任务类型为 `lightweight_runtime_quota_shutdown`。

邮件预警：

- 当剩余运行时长 `<= 2 小时` 时，系统会向该用户的已验证邮箱发送提醒。
- 同一轮预警窗口只会发送一次；管理员上调配额后，如果剩余时间重新回到 2 小时以上，预警标记会自动清空。

兼容说明：

- 功能上线前已经在运行中的轻量云 VM，会从系统首次观察到该 VM 的时刻开始累计，不会追溯历史真实开机时间。

## 清理规则

删除 VM 时会同步清理该 VM 的专属安全组、规则、轻量云 VM 级配额、月流量状态和注册 VM 列表记录。

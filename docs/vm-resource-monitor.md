# 虚拟机资源监控

## 功能概述

虚拟机资源监控提供两方面能力：

1. **列表概览**：虚拟机列表中直接显示每台运行中VM的 CPU 和内存实时使用率
2. **详情历史**：虚拟机详情页支持实时监控和历史记录查询

## 数据采集

### 采集机制

后台启动独立协程自动采集运行中虚拟机的资源数据：

| 项目 | 说明 |
|------|------|
| 采集间隔 | 10 秒 |
| 持久化间隔 | 60 秒 |
| 采集指标 | CPU使用率、内存、网络流量、磁盘I/O |
| 存储方式 | SQLite `vm_stats_records` 表 |

### 采集指标

- **CPU 使用率**：两次采样计算差值（virsh cpu-stats）
- **内存使用**：virsh dommemstat
- **网络流量**：virsh domifstat（接收/发送字节数）
- **磁盘 I/O**：virsh domblkstat（读/写字节数）

## 列表展示

虚拟机列表中新增两列：
- **CPU占用**：进度条样式，颜色随使用率变化（绿→黄→红）
- **内存占用**：进度条样式，同上
- 关机状态显示 `-`

列表概览默认直接使用后台缓存中的实时资源数据，不额外触发单台虚拟机的即时采样。

## 历史查询

在虚拟机详情页点击「历史查询」标签页，选择日期范围后点击查询：

- 支持的日期格式：`YYYY-MM-DD` 或 `YYYY-MM-DDTHH:mm:ss`
- 图表会切换为历史模式，展示所选时间范围内的 CPU、内存、网络、磁盘数据
- 点击「实时监控」可切回实时模式

### API 接口

```
GET /api/vm/:name/stats/history?start=2026-03-01&end=2026-03-17
```

返回格式：
```json
{
  "code": 200,
  "data": [
    {
      "vm_name": "test-01",
      "cpu_percent": 12.5,
      "mem_used": 524288,
      "mem_total": 2097152,
      "net_rx_bytes": 1048576,
      "net_tx_bytes": 524288,
      "disk_rd_bytes": 2097152,
      "disk_wr_bytes": 1048576,
      "recorded_at": "2026-03-17T10:00:00+08:00"
    }
  ]
}
```

## 数据清理

- 删除虚拟机时自动清理该VM的所有历史资源记录
- 无需手动干预

## 环境变量

本功能无新增环境变量，使用已有的数据库配置（`KVM_DB_PATH`）存储数据。

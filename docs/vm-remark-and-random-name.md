# 虚拟机备注与默认随机名称

## 功能说明

本次调整补齐了虚拟机名称和备注的基础体验：

- 新建虚拟机时，表单会默认生成一个随机名称，格式为 `vm` + 8 位字母数字，例如 `vm3k8d2p9`
- 虚拟机名称现在允许使用字母和数字，不再限制为纯字母
- 创建、模板克隆、原生链式克隆、导入磁盘时都支持填写备注
- 备注支持通过独立入口单独修改或清空，不需要进入“编辑虚拟机”表单
- 虚拟机列表和虚拟机详情页会显示备注
- 虚拟机列表的搜索框支持同时搜索名称和备注内容

## 备注存储方式

虚拟机备注存放在 libvirt 的 domain metadata 中，不依赖数据库。

这样做的好处：

- 与现有运行中的虚拟机定义保持同步
- 迁移或脚本化管理时更容易跟随虚拟机一起带走
- 不会引入额外的虚拟机配置表耦合

## 接口变化

以下现有接口新增可选字段 `remark`：

- `POST /api/vm/create`
- `POST /api/self/vm/create`
- `POST /api/vm/clone`
- `POST /api/vm/linked-clone`
- `POST /api/self/vm/clone`
- `POST /api/vm/linked-clone`
- `POST /api/self/vm/import`
- `PUT /api/vm/:name`

其中：

- 创建类接口里的 `remark` 会在虚拟机定义完成后写入 metadata
- 编辑接口里的 `remark` 支持单独更新和清空

## 界面入口

- 虚拟机列表的备注列支持直接点击“编辑备注”
- 虚拟机详情页的备注行支持直接点击“编辑备注”
- “编辑虚拟机”主表单不再承载备注修改，避免与常规资源配置调整混在一起

## 使用建议

- 名称建议继续保持简短稳定，适合作为系统标识，例如 `web01`、`db2026`
- 更详细的业务说明、环境标签、迁移备注建议写在备注中
- 如果后续需要批量查询或批量筛选备注，可以在现有 metadata 基础上继续扩展

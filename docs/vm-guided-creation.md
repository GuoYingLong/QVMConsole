# 虚拟机引导式创建菜单 (Guided VM Creation)

## 功能说明
为了提升创建虚拟机的用户体验，将以前的“全部展开在标签页中”的表单交互方式变更为**步骤分阶段的引导式菜单（Wizard）**。目前，新建虚拟机时用户仅需按照引导一步步完成相关配置，从而避免在面对大量复杂参数时感到困惑。该功能使得配置流程更清晰，同时仍保留了底层原本丰富的功能字段。

### 适用场景
- **普通用户与管理员**通过控制台前端界面新建虚拟机时。
- 注意：由于只更改了前端的交互流程布局，编辑虚拟机仍然保留传统的标签页模式，以便于快速找到对应参数并进行散点式配置修改；“新建”时才强制使用引导流程。

## 交互步骤
根据不同的创建模式，引导步骤自动动态适配：

### 1. 步骤0：选择创建方式 (Mode)
用户首先必须选择创建方式之一：
- **📀 使用 ISO 创建**：使用标准的系统镜像安装全新操作环境；
- **📄 从模板创建**：快速克隆预配置好的系统模板；
- **🔗 原生链式克隆**：仅管理员可见，直接基于模板创建 overlay 并启动，不修改来宾系统；
- **📥 导入虚拟机**：使用已有的虚拟机磁盘文件直接运行。
选择任意一项会自动进入“下一步”。

### 2. 步骤1：基础配置 (Basic)
配置虚拟机的最基础信息，这也是通过“下一步”前唯一受严格表单级校验的步骤：
- **通用**：虚拟机名称 (不可含特殊字符)、CPU、内存。
- **特定模式附加**：
  - ISO模式：系统盘大小、系统类型、系统版本。
  - 导入模式：从存储中选择磁盘文件。
  - 模板模式：选择模板、确认随机主机名、填写用户名与强密码。
  - 原生链式克隆模式：选择模板并确认磁盘大小，不显示登录凭据输入区。
- 模板模式会在选择模板后自动回填模板记录的 CPU、内存、系统盘大小、系统盘驱动、网卡型号、显示设备和 CPU 拓扑模式，并尽量按模板元数据带出引导固件；其中 Windows 的 UEFI 模板会直接显示为“UEFI + 安全引导”。

### 3. 步骤2：硬件与磁盘 / 网络与安全
- **ISO 或 导入模式 (硬件与磁盘)**：选择虚拟化方案（KVM / QEMU）、平台架构、引导类型（BIOS/UEFI）等。
- **模板模式 (网络与安全)**：模板模式会自动生成随机主机名，支持一键生成强密码，并要求用户名/密码在前端和后端同时通过校验后才能提交。
- **原生链式克隆模式 (网络与安全)**：仍可选择 VPC、安全组、存储位置、启动固件、RTC、Guest Agent、SMBIOS 与动态内存，但不会触发任何来宾初始化。

### 4. 步骤3-5：网络与介质、启动与安全、高级设置
主要用于 ISO 或导入模式：
- **网络与介质**：选择 ISO 挂载、网卡类型以及 VPC 交换机/安全组。ISO 模式支持一次选择多个 ISO，首个 ISO 作为主安装盘，其余 ISO 作为额外挂载光驱。挂载 Windows ISO 时，前端会自动推荐更兼容的引导与硬件组合，但不会覆盖用户已经手动改过的引导类型。速率限制已统一移动到虚拟机创建后的「网络管理」中配置。
- **启动与安全**：管理启动项优先级顺序（可拖动），以及 Watchdog、开机自启开关。
- **高级设置**：独立展示开发者选项，当前提供“启动时冻结 CPU”、“CPU 拓扑”、“RTC 配置”、“SMBIOS 类型 1 配置”等入口，并在页面中明确提示该区域属于进阶设置，通常建议保持默认值。

### 关键控制逻辑
- **前端步骤控制框架**：使用基于 Element Plus 的 `<el-steps>` 作为进度指示器。
- **字段依赖验证**：在调用“下一步”从基础配置跳转到下一步骤时，框架将使用 `form.validateField` 局部检验当前重要字段，若未按提示填好名称/CPU/等将无法跳到下一步，提高了表单填写的严谨性。
- **动态步骤计算**：`createSteps` 计算属性根据当前表单中的 `create_mode` 自动构建合适的步骤列表，并利用 `v-show` 切换不同的参数卡片。

## 技术实现
- **文件位置**：`web/src/components/VmForm.vue`
- **样式**：采用简单的淡入 `fadeIn` 动画过渡，使得切换更加自然。将原有的“模式单选按钮”升级为精致的 `mode-card` 交互组件，在鼠标悬停和点击时有高度的响应式视觉反馈。

## 当前约束
- 模板克隆模式新增了用户名与强密码必填规则。
- 原生链式克隆模式不会要求输入用户名、密码或主机名。
- 模板克隆模式的主机名默认不再跟随虚拟机名称，而是自动生成随机值，并允许手动重置。
- 模板克隆模式当前会优先复用模板元数据中的 `boot_type` 作为表单默认引导固件展示，减少手动调整。
- 模板克隆和原生链式克隆会优先复用模板元数据中的 `default_config`，自动带出 CPU、内存、系统盘大小、系统盘驱动、网卡型号、显示设备、CPU 拓扑模式和 Windows 首次重启策略。
- 如果旧模板元数据缺少 `boot_type`，或元数据与模板磁盘实际引导类型不一致，后端会回退检测并自动纠正，避免 UEFI 模板在前端误显示为 `BIOS`。
- 如果旧模板元数据缺少 `default_config`，前端会自动回退到当前默认值，不影响旧模板继续创建。
- 后端接口现已同步校验主机名、用户名和强密码格式，避免绕过前端校验。

# 用户总运行时长配额

## 功能概述

为普通用户新增“总运行时长配额”能力，用于统计该用户名下**所有运行中/暂停中虚拟机的累计运行总时长**。

- 管理员可在“用户管理”中为普通用户设置总运行时长配额，单位为**小时**
- `0` 表示不限
- 统计口径为该用户所有 VM 的总和
  - 例如同时运行 2 台 VM 1 小时，会累计 2 小时
- 达到配额后：
  - 系统会自动关闭该用户当前运行中的全部 VM
  - 后续所有开机操作都会被拒绝
- 当剩余配额不足 2 小时时，会自动发送邮件提醒用户

## 统计口径

### 计时范围

以下状态会计入总运行时长：

- `running`
- `paused`

### 累计方式

系统会在后台按用户维度维护：

- 已累计总运行秒数
- 上次观测时间
- 上次观测时的运行中 VM 数量

每次同步时按以下方式累计：

```text
新增时长 = 距离上次观测的秒数 × 上次观测时的运行中 VM 数量
```

这样可以正确累计“多台虚拟机同时运行”的总时长。

## 重启不中断

与单台 VM 的“连续运行时间”不同，这个功能会将运行时长统计状态持久化到数据库，因此：

- 面板重启/更新后不会归零
- 面板恢复后会继续按上次观测状态累计
- 即使中间重启，用户总运行时长配额仍然连续生效

说明：

- 该功能仍以宿主机 `virsh` 状态探测为准
- 若面板停机期间用户在宿主机侧直接操作 VM，最终累计值可能存在一个采集周期内的轻微偏差

## 开机校验

以下场景都会额外校验总运行时长配额：

- 用户手动开机
- 管理员代用户开机
- 用户通过控制台拉起 VM
- 克隆/创建/导入后自动开机
- 救援模式切换过程中触发的开机

若配额已耗尽，会直接返回提示并拒绝启动：

```text
总运行时长配额已用尽，系统已禁止继续开机
```

## 自动关机

后台资源采集器会持续检查普通用户的总运行时长配额。

当检测到：

- 配额已耗尽
- 且该用户仍有运行中的 VM

系统会自动提交异步任务，关闭该用户名下全部运行中的 VM。

任务类型：

- `runtime_quota_shutdown`

关闭完成后，不会自动恢复开机，需要管理员调整配额后用户再手动开机。

## 邮件预警

当满足以下条件时，会发送配额预警邮件：

- 用户为普通用户
- 用户状态为正常
- 已绑定并验证邮箱
- 系统已配置 SMTP
- 剩余总运行时长 `<= 2 小时`
- 当前预警窗口内尚未发送过提醒

邮件内容会提示：

- 已用时长
- 剩余时长
- 配额上限
- 耗尽后的自动关机与禁开机行为

当管理员重新上调配额，且剩余时间重新回到 2 小时以上后，预警标记会自动清空，后续再次接近阈值时会重新发送提醒。

## 前端展示

以下页面已接入该配额展示：

- 用户管理列表
- 用户配额编辑弹窗
- 新增用户弹窗
- 邀请注册页
- 普通用户 Dashboard“我的资源”

展示字段包括：

- 已用总运行时长
- 配额上限（小时）
- 使用进度
- 是否已耗尽

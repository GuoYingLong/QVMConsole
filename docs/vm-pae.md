# 虚拟机 PAE（物理地址扩展）

## 功能说明

虚拟机高级设置新增了 `PAE`（Physical Address Extension，物理地址扩展）开关，用于控制来宾系统是否暴露 PAE 能力。

该能力主要对以下场景有意义：

- 部分 x86 老系统
- 32 位系统需要识别更大物理内存
- 启动兼容性排查时，需要显式比对是否开启 PAE

默认行为：

- 未显式设置时默认启用
- 非 x86 架构（如 `aarch64`、`riscv64`）会自动忽略该设置，避免写入无效 XML

## 页面位置

- 创建虚拟机向导：`高级设置`
- 编辑虚拟机弹窗：`高级设置`
- 虚拟机详情页：`高级设置` 卡片中可查看当前状态

## 生效方式

- 新建 / 克隆 / 导入 / 轻量云登记：在定义虚拟机 XML 时写入 `<features><pae/></features>`
- 编辑已有虚拟机：写入持久配置，建议关机后修改并重新开机生效

## 使用建议

- 普通 Linux / Windows 虚拟机保持默认即可
- 如果是 64 位系统，通常不需要手动调整
- 如果是老旧 32 位系统、特殊驱动环境或引导兼容性排障，可尝试显式开启或关闭对比
- 若虚拟机已经在运行，建议先关机再改，避免把运行时行为和配置变更混在一起排查

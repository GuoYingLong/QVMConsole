# 端口转发手动宿主机端口占用校验

## 功能说明

当用户在新增或编辑端口转发时手动填写宿主机端口，后端会先做宿主机端口占用校验，再继续后续流程。

校验范围包括：

- 宿主机当前监听中的 TCP/UDP 端口
- 已存在的端口转发规则

## 行为变化

### 新增端口转发

- 如果手动填写的宿主机端口已被占用，接口会直接返回错误
- 发生端口冲突时，不会继续执行自动静态 IP 绑定，也不会写入新的转发规则

### 编辑端口转发

- 如果手动改成的新宿主机端口已被其他监听或其他端口转发占用，接口会直接返回错误
- 如果只是保留当前规则自己的宿主机端口，系统不会把当前规则误判为冲突
- 端口冲突时，不会先删除原规则，避免编辑失败导致原转发被意外移除

## 返回提示

冲突时会返回类似提示：

```text
宿主机端口 10022/tcp 已被占用: 端口已被宿主机监听占用（tcp/10022）
```

或：

```text
宿主机端口 10022/tcp 已被占用: 已存在端口转发规则（tcp/10022 -> 192.168.122.10:22）
```

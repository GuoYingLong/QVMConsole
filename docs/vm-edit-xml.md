# 虚拟机 XML 编辑

## 功能说明

在虚拟机编辑弹窗的“高级设置”中新增“Domain XML -> 编辑 XML”入口，用于直接查看和修改当前虚拟机的持久化 libvirt domain XML。

该功能仅对管理员开放，面向明确知道 libvirt XML 结构的高级用户。保存时会直接调用 `virsh define` 重写持久化定义，因此前端会触发二次验证，并在保存后刷新当前表单。

## 使用方式

1. 进入虚拟机编辑弹窗
2. 打开“高级设置”
3. 点击“Domain XML -> 编辑 XML”
4. 修改文本内容后点击“保存 XML”

## 行为规则

- 读取的是 `virsh dumpxml --inactive` 返回的持久化 XML
- 保存时写回的是持久化配置，不会直接热修改运行态设备
- 虚拟机处于运行中或暂停态时，保存后的改动通常需要关机再启动才会完全生效
- 保存成功后，弹窗会重新拉取 XML，并刷新当前编辑表单与磁盘列表

## 安全限制

- 该接口属于高风险操作，保存时需要完成二次验证
- 该入口与接口仅管理员可用
- 不支持通过 XML 编辑器修改虚拟机名称
- XML 内容必须是合法的 `domain` 根节点结构
- 普通表单里尚未提交的改动，若先保存 XML，会被刷新覆盖

## 适用场景

- 临时添加当前面板尚未提供表单项的 libvirt 参数
- 排查引导、设备模型、通道、features 等底层兼容问题
- 对照现有表单生成结果，做小范围高级调优

## 不建议的场景

- 将其当作日常配置入口
- 在不了解 libvirt domain XML 结构时直接修改
- 在运行中的生产虚拟机上频繁试错

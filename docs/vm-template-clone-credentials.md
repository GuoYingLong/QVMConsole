# 模板创建表单凭据规则

## 适用范围

- 前端页面：`web/src/components/VmForm.vue`
- 后端接口：
  - `POST /api/vm/clone`
  - `POST /api/self/vm/clone`

## 当前行为

### 1. 用户名与密码改为必填
- 在“从模板创建”模式下，用户名和密码现在都必须填写。
- 前端会在表单阶段直接校验，后端也会再次校验，避免绕过页面直接调用接口。

### 1.1 Windows 模板默认用户名
- 当模板类型为 `Windows` 时，用户名会默认填充为 `administrator`。
- 该用户名在前端会同步锁定，后端也会再次强制校验，避免被接口绕过修改。
- 页面会提示 `Windows Server` 请保持默认，不支持修改。

### 2. 主机名改为随机生成
- 进入“从模板创建”模式时，会自动生成类似 `vm-abc123de` 的随机主机名。
- 用户仍可手动修改，也可以点击“随机生成”按钮重新生成。
- 若请求未传主机名，后端会兜底自动生成随机主机名。

### 3. 密码支持一键生成强密码
- 密码输入框后新增“生成强密码”按钮。
- 点击后会生成满足规则的密码，并保证末尾带一个随机符号。

### 4. 模板磁盘大小默认跟随模板
- 在“从模板创建”模式下，选择模板后会自动将磁盘大小填充为该模板的虚拟磁盘大小。
- 用户仍可手动扩大磁盘，但不能填写小于模板原始磁盘大小的值。
- 后端接口会再次按模板实际磁盘大小做兜底修正，避免绕过前端后传入过小值。

## 强密码规则

- 长度至少 `12` 位
- 必须同时包含：
  - 大写字母
  - 小写字母
  - 数字
  - 符号
- 为兼容当前克隆初始化脚本与命令注入安全限制，仅允许以下符号：
  - `!@#$%^&*_-+=?`

## 格式限制

### 用户名
- 只能以小写字母或下划线开头
- 只能包含小写字母、数字、下划线和短横线

### 主机名
- 只能包含字母、数字和短横线
- 不能以短横线开头或结尾

## 说明

- `linux` 模板会在克隆后通过 SSH 执行系统内初始化。
- `windows` 模板当前仅负责创建虚拟机，不会自动创建网页登录账号。
- `fnos` 模板会在开机前离线注入首个管理员账号、主机名和初始化标记：
  - 表单中的“用户名”会作为 `fnOS` 首次网页登录管理员账号。
  - 表单中的“密码”会作为该管理员账号的网页登录密码。
  - 若用户名留空，后端会默认注入 `admin` 账号。
- `fnos` 模板在注入首次登录信息前，会先离线扩展最后一个 ext 系统分区；该流程不依赖 SSH，详细说明见 `docs/fnos-template-disk-expand.md`。
- `fnos` 模板新增“设备 ID”单选项：
  - 默认“重新生成”：保持原有初始化行为，离线清空 `/etc/machine-id` 并删除 `/var/lib/dbus/machine-id`，首次启动后由系统和 FnOS 服务生成新的系统标识与设备 ID。
  - 选择“保留设备 ID”：不清空模板内 `/etc/machine-id`，并在缺少 `/var/lib/dbus/machine-id` 时用 `/etc/machine-id` 补齐；模板中的 `/usr/trim/etc/machine_id` 与 `/etc/device_id` 会随链式克隆保留，适合需要克隆后飞牛设备 ID 完全一致的场景。
  - 选择“指定设备 ID”：额外填写 `32` 位或 `40` 位十六进制 ID，后端会写入 `/etc/device_id` 与 `/usr/trim/etc/machine_id`；若填写 `32` 位，`/usr/trim/etc/machine_id` 会使用该值加 `00000000` 补足为 `40` 位。
- 接口字段为 `preserve_fnos_device_id` 和 `fnos_device_id`，仅对 `template_type=fnos` 的模板克隆生效；传入 `fnos_device_id` 时会自动按“指定设备 ID”处理。原生链式克隆本身不执行初始化，仍会天然保留模板内设备 ID。
- 该流程不依赖 SSH，因此不会使用模板中的 root 密码做在线初始化。
- 该流程复用既有 `libguestfs-tools` 依赖，没有新增 apt 或第三方依赖。
- 该规则当前只针对"从模板创建"单台虚拟机的表单与接口生效，不影响其他创建流程。

## 克隆模式（新增）

在"从模板快速克隆"和"原生链式克隆"创建方式下，新增 **克隆模式** 选项：

- **链式克隆（默认）**：`clone_mode=linked`
  - 使用 `qemu-img create -f qcow2 -F qcow2 -b <模板> <目标>` 创建 backing_file 链式磁盘
  - 依赖模板存在，磁盘创建速度快，节省存储空间
  - **注意**：如果模板被删除，所有链式克隆的虚拟机将无法启动

- **完整克隆**：`clone_mode=full`
  - 使用 `qemu-img convert -f qcow2 -O qcow2 <模板> <目标>` 将模板数据完整复制到独立磁盘
  - 不依赖模板，脱离链式条件
  - 磁盘创建较慢，占用完整磁盘空间
  - 后续的来宾初始化流程（SSH 配置、磁盘扩容等）保持不变
  - **删除模板时不会关联该虚拟机**：完整克隆完成后，磁盘已独立，不会出现在模板删除预览的"关联虚拟机"列表中

### 接口字段

所有克隆相关接口均已支持 `clone_mode` 字段：
- `POST /api/vm/clone` — `clone_mode: "linked" | "full"`
- `POST /api/vm/linked-clone` — `clone_mode: "linked" | "full"`
- `POST /api/self/vm/clone` — `clone_mode: "linked" | "full"`
- 轻量云注册记录 — `clone_mode: "linked" | "full"`

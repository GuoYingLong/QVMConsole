# 网络管理使用文档

## 功能概述

网络管理模块提供虚拟机网络相关的配置功能，包括端口转发、静态 IP、VPC/安全组、速率限制、运行状态查看和管理员网络诊断。
通过虚拟机详情页的「网络管理」标签页或虚拟机列表的「网络管理」按钮打开。新平台默认基于 OVS `br-ovs` 内网运行，DHCP 租约与静态 IP 由 `/var/lib/kvm-console/ovs/dnsmasq.leases` 和 `/etc/kvm-console/ovs/dhcp-hosts` 管理。

## 端口转发

### 配额与使用情况

- 普通用户默认最多可创建 **10 条** 端口转发规则
- 管理员可在「用户管理 → 配额」中调整该上限
- 端口转发页会显示当前用户 **已用数量 / 配额上限**
- 配额设置为 `0` 表示不限

### 添加转发

点击「添加转发」按钮，填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 宿主机端口 | 否 | 留空则自动从配置范围内分配空闲端口 |
| 虚拟机端口 | 是 | 需要转发到的虚拟机端口（如 22、80） |
| 协议 | 否 | 默认 TCP，可选 UDP 或 TCP+UDP |

**自动化行为：**
- **VM IP 自动获取**：无需手动输入虚拟机 IP，系统自动从静态绑定中获取
- **自动绑定静态 IP**：如果虚拟机没有静态 IP 绑定，系统会自动分配一个空闲 IP 并绑定
- **自动分配宿主机端口**：如果宿主机端口留空，系统从配置的端口范围内自动分配
- **手动端口前置校验**：如果用户手动指定宿主机端口，后端会先检查宿主机监听和现有端口转发占用；若端口冲突，会直接返回错误，不继续做静态 IP 绑定或写入转发规则
- **自动补安全组**：VM 已绑定 VPC 安全组时，会自动补一条对应协议和内部端口的入站允许规则
- **宿主机防火墙联动**：无论宿主机防火墙当前是否启用，系统都会给端口转发补 UFW 放通规则，避免下次开启宿主机防火墙后拦截已有转发
- **自动持久化与恢复**：添加/删除规则后自动保存到 `/etc/kvm-portforward/rules.sh`；服务启动时会读取该文件恢复端口转发规则。保存脚本使用幂等的 `iptables -C ... || iptables -A ...` 格式，避免服务重启时重复追加。非 VPC 目标会保存传统 FORWARD 放行，VPC 目标只保存 DNAT，放行由安全组 ACL 决定
- **完整访问地址展示**：系统设置中的「公网 IP」可指定端口转发对外展示和规则优先使用的公网地址；端口转发列表会显示完整访问地址，例如 `203.0.113.10:10022`，并支持一键复制（HTTP 场景会自动使用降级复制方案）

## VPC / 安全组

「VPC / 安全组」标签页显示当前 VM 绑定的交换机、交换机子网、安全组和归属用户。

| 操作 | 说明 |
|------|------|
| 保存 VPC 绑定 | 更新 VM 当前交换机和安全组，会同步更新运行中 OVS VLAN tag 与 ACL |
| 仅切换安全组 | 只重建 ACL，不修改网卡 |
| 绑定 IP | 为已绑定 VPC 的 VM 写入当前交换机的静态 DHCP 绑定 |

旧 VM 纳入 VPC 时建议先关机再调整网络，避免运行中变更网卡导致断网。

如果 VM 正在运行，保存 VPC 绑定不会进入虚拟机内部释放旧 DHCP 租约。宿主机侧 VLAN 已切换后，虚拟机仍可能继续显示原 IP；需要在虚拟机内重新获取 DHCP、重启网卡，或重启虚拟机后才会显示新 VPC 子网地址。

VM 级速率限制默认使用 `0`，表示单台 VM 不额外限速；外网聚合带宽和月流量由 VPC 交换机控制。只有需要对单台 VM 单独收紧上下行速率时，才在 VM 网络管理中设置非 `0` 的平峰速率。已绑定 VPC 的 VM 不参与用户带宽配额自动均分，避免覆盖交换机级策略。

### 删除转发

点击对应规则的「删除」按钮即可。同时会自动删除非 VPC FORWARD 规则；若该转发曾由面板自动创建 UFW 放行规则，系统会同步清理这条面板管理规则。VPC 转发由安全组 ACL 控制。若该转发曾自动创建安全组入站放行规则，且没有其它端口转发继续使用同一个目标 IP、协议和内部端口，系统会同步删除备注为「端口转发自动放行」的安全组规则；用户手动创建或修改备注的规则不会被自动删除。

### 批量删除

- 端口转发列表支持多选后批量删除
- 批量删除和单条删除一样，都会触发高风险二次验证
- 删除完成后页面会自动刷新数量占用情况

### 编辑转发

- 端口转发列表支持对单条规则进行编辑
- 可调整目标 IP、宿主机端口、虚拟机端口和协议
- 手动修改宿主机端口时，会先检查该端口是否已被宿主机监听或其他端口转发占用；冲突时不会先删除旧规则
- 编辑成功后会保留原有的入站区域限制豁免状态

## 静态 IP 管理

### 绑定静态 IP

点击「绑定 IP」按钮：

| 字段 | 必填 | 说明 |
|------|------|------|
| 虚拟机名称 | 是 | 自动填充为当前虚拟机名称 |
| IP 地址 | 否 | 留空自动分配空闲 IP；可输入完整 IP 或最后一位数字（如输入 `10` 等同于 `192.168.122.10`） |

**自动化行为：**
- **自动分配 IP**：IP 留空时，从 `.2` 到 `.254` 按顺序查找第一个空闲 IP（排除已有静态绑定和 DHCP 租约）
- **VPC 子网适配**：VM 已绑定 VPC 时，静态 IP 写入当前交换机独立的 DHCP hosts 文件，并从该交换机 DHCP 地址池分配；输入最后一位数字时按交换机网关所在网段补全
- **保留当前地址**：VM 已绑定 VPC 且当前已有 DHCP/邻居表地址时，自动绑定会优先把当前正在使用的 IP 固化为静态 IP，避免运行中 VM 因另分配地址而短暂变更
- **网卡自动刷新**：绑定后如果虚拟机正在运行，VPC 静态 IP 会先快速写入 DHCP 配置并后台刷新网卡；虚拟机内部仍可能需要重新获取 DHCP 或重启网络后才显示新 IP
- **网关保护**：`.1` 固定为网关，不会被分配

### 解绑静态 IP

点击对应绑定的「解绑」按钮。如果虚拟机正在运行，同样会自动执行网卡拔插，使虚拟机切换回 DHCP 动态 IP。

### DHCP 租约

显示当前活跃的 DHCP 租约信息（已过滤掉与静态绑定重复的条目）。

## 速率限制

「速率限制」标签页用于设置当前 VM 的外网平峰带宽：

| 字段 | 说明 |
|------|------|
| 下行速率 | 流量进入 VM 的持续带宽限制，单位 Mbps |
| 上行速率 | 流量离开 VM 的持续带宽限制，单位 Mbps |

普通用户可自行调整自己 VM 的速率限制，但后端会校验：

- 单台 VM 的下行/上行不能超过用户对应方向的带宽配额。
- 同一用户所有 VM 的下行/上行平峰速率总和不能超过用户对应方向的带宽配额。
- 用户某个方向存在有限配额时，该方向不能设置为 `0`，避免绕过配额。
- 用户日流量超限时，保存后仍会按现有惩罚速率生效。

管理员设置速率限制不受用户配额限制；如果下行和上行都设置为 `0`，系统会优先按 VM 归属用户重新分配带宽，找不到归属用户时清除该 VM 的速率限制。

## IP 获取机制

虚拟机 IP 地址在列表页和详情页都会显示，获取优先级如下：

1. **QEMU Guest Agent**（最准确，需虚拟机安装 `qemu-guest-agent`）
2. **OVS DHCP 租约**（通过 dnsmasq 租约文件查询）
3. **ARP 表**（通过 ARP 缓存获取）
4. **静态绑定**（从网络 XML 配置中读取，显示为 `x.x.x.x (静态)`）

> 无论虚拟机是否运行，都会尝试获取 IP。关机状态的虚拟机如果有静态绑定，会显示静态绑定的 IP。

## 网络诊断

管理员可在「网络诊断」标签页对当前 VM 的运行态 `vnet*` / `tap*` 接口执行临时抓包。

| 功能 | 说明 |
|------|------|
| 抓包接口 | 只允许选择当前 VM 关联的运行态接口 |
| 过滤条件 | 支持协议、源 IP、目标 IP、任意端口、源端口、目标端口 |
| 快捷模板 | 提供 ARP、DHCP、DNS、当前 VM IP 和端口转发入站流量模板 |
| 实时摘要 | 页面轮询显示 tcpdump 文本输出 |
| pcap 下载 | 任务完成后可下载 pcap 文件 |
| pcap 删除 | 可删除当前 pcap；新抓包开始前会自动删除上一份 pcap |

抓包属于高风险诊断操作，仅管理员可用，提交前需要二次验证。该功能不修改虚拟机配置、不改 OVS 流表、不重启网络服务。详细说明见 `docs/network-diagnostics.md`。

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/network/static-ip/list` | 获取静态 IP 列表和 DHCP 租约 |
| POST | `/api/network/static-ip/bind` | 绑定静态 IP（ip 可留空） |
| POST | `/api/network/static-ip/unbind` | 解绑静态 IP |
| GET | `/api/network/port-forward/list` | 获取端口转发列表 |
| POST | `/api/network/port-forward/add` | 添加端口转发（自动获取 IP、自动绑定、自动分配端口） |
| PUT | `/api/network/port-forward/:id` | 编辑单条端口转发 |
| DELETE | `/api/network/port-forward/:id` | 删除端口转发 |
| POST | `/api/network/port-forward/batch-delete` | 批量删除端口转发 |
| GET | `/api/vm/:name/network/status` | 获取 VM 网络运行状态和当前速率限制 |
| GET | `/api/vm/:name/network/diagnostics` | 获取 VM 网络诊断摘要和抓包模板 |
| POST | `/api/vm/:name/network/capture` | 提交 VM 网络抓包任务，需要二次验证 |
| GET | `/api/network/captures/:task_id` | 获取抓包任务摘要和文件状态 |
| GET | `/api/network/captures/:task_id/download` | 下载抓包 pcap 文件 |
| DELETE | `/api/network/captures/:task_id` | 删除抓包 pcap 文件 |
| PUT | `/api/vm/:name` | 更新 VM 速率限制（仅传 `bandwidth_inbound_avg` / `bandwidth_outbound_avg`） |

### 添加端口转发请求参数

```json
{
  "vm_name": "clone",      // 必填：虚拟机名称
  "vm_port": "22",          // 必填：虚拟机端口
  "host_port": "",          // 可选：留空自动分配
  "protocol": "tcp"         // 可选：tcp/udp/both，默认 tcp
}
```

### 绑定静态 IP 请求参数

```json
{
  "vm_name": "clone",       // 必填：虚拟机名称
  "ip": ""                  // 可选：留空自动分配
}
```

### 更新速率限制请求参数

```json
{
  "bandwidth_inbound_avg": 50,
  "bandwidth_outbound_avg": 10
}
```

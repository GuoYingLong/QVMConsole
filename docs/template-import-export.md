# 模板导入导出

## 功能概述

模板导出统一生成 `.tar.gz` 模板包，包内包含：

- `manifest.json`
- 每个模板节点的 `.qcow2`
- 每个模板节点的 `.meta.json`

导入时先解析模板包并展示完整链路，管理员确认后才提交导入任务。

## 导出范围

- 在根模板标题导出：导出整棵模板树
- 在某个节点导出：导出该节点及其所有子节点

导出任务通过任务队列执行，完成后可在任务中心下载 `.tar.gz` 包。

模板管理页会在每个节点展示导出包状态。已导出的节点可以直接下载导出包，也可以删除当前节点对应的导出包；重新导出会覆盖该节点上一次生成的导出包。

页面位置：模板管理页每个节点右侧操作区会展示“下载导出包”和“删除导出包”按钮，仅在该节点已有导出包时显示。

## 导入预览

导入页面只展示模板包信息，不允许补全或修改元数据。

预览内容包括：

- 模板树唯一标识
- 节点链路
- 管理员名称
- 用户侧显示文本
- 启用状态
- 禁用状态
- MD5 / SHA256 / 文件大小
- 新增、已存在或冲突状态

## 增量更新

系统通过 `template_uid` 判断导入包是新模板树还是已有模板树：

- `template_uid` 不存在：作为新模板树导入
- `template_uid` 已存在：进入更新流程

更新流程只新增包内新节点，不覆盖已有节点。

导入前会校验本地已有节点的：

- 文件大小
- MD5
- SHA256

任一节点不匹配时会拒绝更新，并提示对应节点。

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/template/import/preview` | 上传或指定 `.tar.gz` 包，返回链路预览 |
| POST | `/api/template/import/confirm` | 按预览 token 确认导入 |
| POST | `/api/template/:name/export?scope=node` | 导出当前节点子树 |
| POST | `/api/template/:name/export?scope=root` | 导出根模板整棵树 |
| GET | `/api/template/download/:filename` | 下载导出的模板包 |

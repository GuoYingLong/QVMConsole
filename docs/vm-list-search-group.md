# 虚拟机列表搜索与分组功能

## 1. 功能概述

虚拟机列表页面包含以下功能：

1. **搜索包含备注**：搜索框同时搜索虚拟机名称和备注字段。
2. **分组视图（默认）**：列表默认按运行状态分组展示，可通过按钮组切换视图模式。
3. **自定义分组**：用户可为每台虚拟机分配自定义分组标签，然后按自定义分组查看。

## 2. 搜索包含备注

- 搜索框提示文字为「搜索名称/备注」
- 同时匹配 `name` 和 `remark` 字段，大小不敏感
- 搜索结果在所有视图下均生效

## 3. 视图切换

筛选栏右侧按钮组提供四种模式：

| 按钮 | 说明 |
|---|---|
| **按状态**（默认） | 运行中 / 已暂停 / 已关机 / 迁移中 |
| **按模板** | 按创建来源模板名称分组 |
| **自定义** | 按用户分配的自定义分组标签分组 |
| **全部** | 传统平铺列表，保留分页 |

## 4. 自定义分组

### 4.1 工作原理

每台虚拟机可以设置一个「分组名称」。分组名称通过 libvirt domain metadata 持久化存储（与备注使用相同的元数据机制），不依赖数据库为单一事实来源。

- 没有设置分组的虚拟机会归类到「未分组」
- 分组名称由用户自由定义，如「生产环境」「开发测试」「数据库」等
- 已有分组会在编辑弹窗中显示，支持快速选择复用

### 4.2 编辑入口

- 列表的「备注/分组」列中，每行有「分组」按钮
- 点击后弹出分组编辑弹窗，可：
  - 输入新的分组名称
  - 点击已有分组标签快速选择
  - 清空输入框可移除分组

### 4.3 分组展示

选择「自定义」视图后，虚拟机按分组名称分组显示：
- 各分组按名称字母序排列，「未分组」始终排在最后
- 每个分组为可折叠面板，功能与状态/模板分组一致

### 4.4 API 接口

通过现有的 `PUT /api/vm/:name` 接口设置分组：

```json
{ "group": "生产环境" }
```

## 5. 分组面板

- **头部**：展开/折叠箭头、分组标签、数量统计、「全选本组」按钮
- **主体**：该组下虚拟机表格，列定义与操作一致
- 分组模式下隐藏分页

## 6. 后端存储

- `vmConfigMetadata` 结构体新增 `Group` 字段，存储于 libvirt domain metadata
- `VmInfo` 新增 `group` 字段（JSON 字段名 `group`）
- `VMCache` 新增 `group_name` 列进行缓存投影
- `GetVMGroup` / `SetVMGroup` 函数遵循 `GetVMRemark` / `SetVMRemark` 相同的实现模式

## 7. 响应式布局

| 屏幕 | 分组视图 | 全部视图 |
|---|---|---|
| > 768px | 分组面板 + 内部表格 | 传统表格 + 分页 |
| ≤ 768px | 分组面板 + 内部卡片 | 移动端卡片列表 |

按钮组在移动端自动占满整行宽度。

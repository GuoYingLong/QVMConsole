# 轻量云 VM 注册确认流程

## 适用场景

轻量云 VM 注册用于“管理员先登记服务器配置，用户确认后再开通”的流程。管理员不再直接分配已有 VM，而是基于模板登记待开通配置；用户登录面板后补全登录凭据，系统再提交任务队列创建服务器。

## 管理员操作

1. 进入用户管理，创建或编辑轻量云用户，并选择专用 NAT VPC。
2. 在创建轻量云用户时点击“添加注册VM”，或在用户列表中点击该用户的“注册VM”。
3. 在注册表单中选择模板并填写 VM 名称、硬件规格、存储位置、系统配置和网络配额。
4. 确认后注册项进入待确认列表；保存后系统会给用户发送邮件。

用户列表中的“注册VM”窗口会同时显示待确认、开通中、失败和已开通的轻量云 VM。管理员可以在该窗口编辑单台 VM 的下行月流量、上行月流量、下行带宽、上行带宽、端口转发上限和快照上限：

- 待确认或失败的 VM：只更新注册记录，用户确认开通时会使用新配置。
- 已开通的 VM：同步更新 VM 运行配额，并立即重新应用带宽限制。
- 开通中的 VM：为避免任务执行期间配置漂移，暂不允许编辑。
- 已开通的 VM：可在操作列点击“移除”，用于移除注册列表记录和单 VM 配额，不删除虚拟机本体。

注册表单复用新建虚拟机表单，但在轻量云注册模式下有以下差异：

- 只允许模板克隆。
- 保留“存储介质”步骤，可选择虚拟机硬盘的目标存储位置。
- 自动使用该轻量云用户的专用 VPC。
- 管理员不选择安全组，开通时系统自动创建或绑定专属安全组。
- 管理员不填写 VM 登录凭据。
- 表单额外显示下/上行月流量、下/上行带宽、端口转发上限和快照上限。

## 用户确认

用户登录后进入虚拟机列表，可以看到“待开通服务器”卡片。用户逐台点击“确认开通”，补全登录凭据后提交。

凭据规则：

- Linux/fnOS：用户填写用户名和强密码。
- Windows：用户名固定为 `administrator`，用户填写强密码。
- 强密码至少 12 位，并包含大小写字母、数字和特殊字符。
- 邮件不会包含 VM 密码。

确认提交会使用现有高风险验证机制；验证通过后系统提交 `lightweight_vm_provision` 任务。

## 开通结果

任务成功后系统会自动完成：

- 从模板克隆并创建 VM。
- Linux 模板会通过 SSH 初始化 hostname、登录用户、登录密码和磁盘扩容；初始化后必须能使用用户确认时填写的新凭据登录，否则开通任务会失败。
- 写入用户 VM 归属。
- 写入轻量云单 VM 流量、带宽、端口转发和快照配额。
- 绑定用户专用 VPC。
- 创建或绑定该 VM 的专属安全组。
- 保存用户确认时填写的登录凭据。

如果克隆、Linux SSH 初始化或某个后置步骤失败，注册项会进入失败状态并保留错误信息，用户可以重新确认开通；若上次失败已经创建出同名 VM，重试时会优先复用该 VM 继续初始化和后置绑定，避免因为“虚拟机已存在”卡住。管理员也可以删除未进入开通流程的注册项。

## 注意事项

- VM 名称仍遵循当前项目规则，只允许纯英文字母。
- 注册项本身不保存用户待填写的 VM 密码。
- 已开通的注册项不会再出现在用户待确认列表。
- 已开通 VM 被删除后，系统会同步删除对应注册项，用户管理中的“注册VM”列表不再保留这台 VM。
- 手动移除已开通 VM 只清理轻量云注册与单 VM 配额数据，不执行虚拟机删除。
- 创建和开通属于耗时操作，均通过任务队列执行。

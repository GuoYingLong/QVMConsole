# 端口转发配额与编辑

## 功能概述

本次更新为用户资源配额新增了 **端口转发数量限制**，并为端口转发页面补充了 **使用情况展示、单条编辑、批量删除** 能力。

## 配额规则

- 普通用户新增字段：`端口转发数量`
- 默认值：`10`
- `0` 表示不限
- 统计口径：当前系统中归属于该用户的端口转发规则总数
- 管理员不受该限制影响
- 轻量云用户不使用用户级端口转发数量配额，端口转发上限在管理员分配 VM 时按单台 VM 设置

说明：

- 使用 `TCP+UDP` 创建转发时，会生成两条独立规则，因此会占用 **2** 个数量配额
- 配额校验在新增端口转发时执行
- 轻量云用户创建、编辑端口转发时，只统计当前 VM 的转发数量；达到该 VM 上限后拒绝新增或扩展为更多协议规则

## 页面变化

### 用户管理

- 新增用户弹窗新增「端口转发」配额输入项，默认 `10`
- 编辑配额弹窗新增「端口转发」配额输入项
- 用户列表新增「端口转发」使用情况列

### 用户首页

- Dashboard 的「网络资源配额」分组新增端口转发卡片

### 虚拟机网络管理

- 端口转发标签页新增：
  - 当前用户已用 / 上限展示
  - 批量删除按钮
  - 单条编辑按钮

### 网络中心

- 管理员端口转发列表新增：
  - 归属用户展示
  - 关联 VM 展示
  - 批量删除
  - 单条编辑

## 交互说明

### 单条编辑

- 支持编辑宿主机端口、虚拟机端口、目标 IP 和协议
- 编辑后会自动刷新列表
- 若原规则配置了“入站区域限制豁免”，编辑后会自动继承

### 批量删除

- 支持在端口转发列表中勾选多条规则后一次删除
- 批量删除仍会触发高风险二次验证

## 接口变更

| 方法 | 路径 | 说明 |
|------|------|------|
| PUT | `/api/network/port-forward/:id` | 编辑单条端口转发 |
| POST | `/api/network/port-forward/batch-delete` | 批量删除端口转发 |

## 权限说明

- 非管理员只能查看和操作自己虚拟机对应的端口转发
- 轻量云用户只能查看和操作自己 VM 的端口转发，且端口转发入口位于 VM 详情页
- 非管理员查询手动 IP 映射时必须指定自己的 VM
- 后端会再次校验 VM 归属，不能只依赖前端页面限制

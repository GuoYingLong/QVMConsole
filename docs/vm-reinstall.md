# 虚拟机重装系统

## 功能范围

重装系统仅面向弹性云 VM。该流程会保留当前虚拟机的 CPU、内存、网络、VPC、安全组、备注、配额、额外数据盘和原有系统盘存储位置，只替换第一块系统盘。

轻量云 VM 不支持使用该功能。

## 请求参数

接口：`POST /api/vm/:name/reinstall`

请求体支持以下字段：

- `template`：目标模板名称，必填
- `disk_size`：重装后的系统盘大小（GB），可选
- `hostname`：重装后的主机名
- `user`：登录用户名
- `password`：登录密码
- `preserve_fnos_device_id`：FnOS 是否保留模板中的设备 ID
- `fnos_device_id`：FnOS 自定义设备 ID，支持 32 位或 40 位十六进制字符串

## 磁盘大小规则

- 未传 `disk_size`：默认使用当前系统盘大小
- 当前系统盘小于模板原始磁盘大小：默认值自动提升到模板最小值
- 显式传入小于模板最小值：前端会阻止提交，后端也会再次兜底提升
- 允许把系统盘从当前更大容量缩小到不小于模板最小值的容量
- 重装不会切换系统盘存储池，也不会修改现有系统盘总线类型

## 执行流程

1. 校验虚拟机未处于迁移中，并检查同一台 VM 是否已有进行中的重装任务
2. 校验模板存在、当前用户有权限使用该模板
3. 校验模板启动族与当前 VM 一致，当前仅允许 BIOS 对 BIOS、UEFI 对 UEFI
4. 自动删除该 VM 的全部快照
5. 直接使用强制关机模式关闭虚拟机
6. 将原系统盘重命名为临时备份
7. 在原路径基于目标模板重新创建新的 qcow2 overlay
8. 按模板类型重新执行初始化
9. 启动原虚拟机并继续完成首启流程
10. 更新模板来源元数据、保存新凭据，并删除旧系统盘备份

## 模板初始化行为

### Linux

- 离线清理 `machine-id`、DHCP 租约和 cloud-init 实例缓存
- 离线写入新的主机名
- 开机后等待 IP，并继续执行 SSH 初始化
- SSH 初始化会处理主机名、用户名、密码和系统盘在线扩容

### Windows

- 离线扩容系统分区
- 重新注入 `unattend.xml`
- 复用首次冷重启策略

### FnOS

- 离线扩容系统分区
- 重新写入首次管理员账号、密码、主机名
- 支持重新生成、保留模板设备 ID，或写入自定义设备 ID

## 回滚说明

如果创建新系统盘、离线初始化、启动或首启初始化任一步骤失败，系统会尽最大努力执行回滚：

- 关闭当前 VM
- 删除新的系统盘
- 恢复原系统盘到原始路径
- 恢复临时修改过的持久化 XML

回滚完成后，虚拟机会保持关机状态，方便管理员继续排查。

## 二次验证与 API Key

- 重装系统属于高风险操作，必须通过二次验证
- 使用 JWT 调用时，收到 `428` 后先完成高风险验证，再携带 `X-High-Risk-Token`
- 使用 API Key 调用时，同样需要补充 `X-High-Risk-Token`

## 使用建议

- 如模板与当前虚拟机的启动族不一致，请先手动新建或迁移到兼容模板，不建议强行重装
- 若模板需要依赖 SSH、`virt-customize`、`guestfish` 等工具完成初始化，请先确认宿主机依赖完整
- 若 VM 存在重要快照，请提前自行导出或记录；重装前会自动清空全部快照

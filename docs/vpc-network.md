# VPC 网络与安全组

## 功能概述

VPC 网络用于把普通用户的虚拟机接入独立逻辑交换机。底层继续复用 OVS `br-ovs`，每个交换机通过 VLAN、独立子网、独立 DHCP、NAT、ACL 和交换机总带宽策略实现隔离。

普通用户创建虚拟机前必须先有交换机，并在创建、克隆或导入虚拟机时选择交换机和安全组。系统会在用户注册激活后自动创建默认安全组和一个「默认交换机」；历史用户访问 VPC 或创建 VM 前仍会幂等补齐默认安全组。

## 页面入口

| 角色 | 入口 | 能力 |
|------|------|------|
| 普通用户 | 左侧「VPC 网络」 | 管理自己的交换机和安全组策略 |
| 管理员 | 左侧「网络」 | 查看 OVS 概览，管理所有用户交换机、安全组、端口转发和 VPC ACL |

## 交换机

创建交换机时需要填写：

| 字段 | 说明 |
|------|------|
| 名称 | 用户内唯一，空格会归一化为 `-` |
| 下行月配额 | 该交换机外网下行月流量配额。用户下行流量总配额为 `0` 时可填 `0` 表示不限；用户总配额有限时必须大于 `0` |
| 上行月配额 | 该交换机外网上行月流量配额。用户上行流量总配额为 `0` 时可填 `0` 表示不限；用户总配额有限时必须大于 `0` |
| 下行总带宽 | 该交换机所有 VM 外网下行聚合带宽上限。用户下行带宽总配额为 `0` 时可填 `0` 表示不限；用户总配额有限时必须大于 `0` |
| 上行总带宽 | 该交换机所有 VM 外网上行聚合带宽上限。用户上行带宽总配额为 `0` 时可填 `0` 表示不限；用户总配额有限时必须大于 `0` |

交换机外网流量按月统计。跨交换机或同交换机内网流量不计入外网配额，但会经过 VPC ACL 与安全组策略。

多个交换机的流量配额和带宽配额会按方向分别累加。用户某个方向总配额有限时，交换机该方向合计不能超过用户配额；用户某个方向总配额为 `0` 时视为不限，管理员账号自身也按同样规则可创建不限流量或不限带宽的交换机。

新用户开通时创建的「默认交换机」会直接使用用户当前配额：用户某方向总配额为 `0` 时，默认交换机该方向也为 `0`，表示不限；用户某方向总配额有限时，默认交换机会先占用该方向全部可用配额。用户需要多个交换机时，可以先编辑默认交换机释放部分配额，再创建其他交换机。

普通用户创建、克隆或导入 VM 时，如果前端请求没有带 `switch_id` 或 `security_group_id`，后端会自动解析默认资源：只有一个交换机时使用该交换机；多个交换机中存在「默认交换机」时使用它；安全组优先使用默认安全组。多个交换机且没有默认交换机时仍要求用户显式选择，避免接入错误网络。

VM 创建、克隆和导入流程会在虚拟机 XML 定义并启动之前写入 VPC 交换机 VLAN，并确保对应交换机运行时配置已就绪。这样虚拟机首次 DHCP 获取地址时就会进入 VPC 子网，不需要等任务完成后的运行时绑定再刷新网卡。

VPC VM 绑定或解绑静态 IP 时会刷新运行中网卡，刷新使用带 VLAN 的 OVS 网卡 XML，并在 attach 成功后重新应用 VPC 运行态，避免 live OVS 端口丢失 VLAN 后短暂进入默认 `192.168.122.0/24` 网段。若历史数据缺少 `vpc_vm_bindings` 记录，后端会优先从 VM 持久 XML、运行中 OVS 端口 tag、VPC 租约和静态绑定文件反推交换机归属，并把误写到基础 OVS DHCP hosts 的 VPC 子网静态 IP 迁回对应交换机。VM 详情页显示 IP 时也会优先使用 VPC 静态绑定或 VPC 租约，并过滤掉不属于当前交换机子网的旧地址。

如果交换机本月下行或上行达到月配额，系统只会把超限方向的生效带宽强制降为 `1Mbps`，未超限方向继续按交换机总带宽策略运行。管理员可在「网络」-「交换机」中点击「重置流量」把当前已用量作为新的计数基线；如果交换机正在被强制限速，重置后会立即解除。若管理员把交换机月配额调大到高于当前已用量，也会自动解除对应方向限速；否则默认在下个月统计窗口重置时恢复正常带宽。

交换机带宽是该交换机下所有 VM 的外网聚合上限。VM 级带宽默认 `0`，表示不额外限制单台 VM；用户只在需要对某台 VM 做更精细的上下行限制时才单独设置 VM 级带宽，且不能绕过交换机聚合上限。已绑定 VPC 的 VM 不参与用户带宽配额自动均分，避免自动给单台 VM 写入默认限速。VM 级下行/上行都保存为 `0` 时，系统会清除该 VM 的 libvirt、OVS 和 TC 单机限速，由 VPC 交换机总带宽继续控制聚合速率。

每个 VPC 交换机运行态会创建独立网关口（如 `vpcsw1`），并为交换机子网写入出网 NAT、出站 FORWARD 与回程 FORWARD 规则。服务启动时会幂等恢复所有已存在交换机的运行态；删除交换机时会同步清理这些规则。若启动恢复发现数据库中仍有 VPC 绑定，但 libvirt 已明确返回该 VM domain 不存在，系统会清理这条残留绑定并跳过该 VM，不再把历史残留误报为 VPC 网络恢复失败；如果是 libvirt 连接失败等其他错误，仍会保留警告以便排查宿主机环境。

交换机上行限速按当前绑定 VM 的 OVS ofport 写入同一个 meter，避免 access port 场景下单纯匹配 VLAN 元数据不命中；交换机下行限速从 VPC 网关端口进入方向写入 meter。VM 绑定、切换交换机或启动后会刷新交换机限速规则。

## 安全组策略

默认安全组采用云厂商风格：

- 默认拒绝入站。
- 默认允许出站。
- 不同交换机默认隔离。
- 规则均表示“允许”，支持入站和出站方向字段。
- 页面中点击「创建安全组」会打开空白表单；点击「编辑」才会回填当前安全组，关闭弹窗后会自动清空表单状态，避免误把新建提交到编辑接口。

安全组规则支持：

| 规则字段 | 说明 |
|------|------|
| 方向 | `ingress` 入站，`egress` 出站 |
| 协议 | `tcp`、`udp`、`icmp`、`all` |
| 端口范围 | TCP/UDP 需要填写，ICMP/全部协议忽略端口 |
| 目标类型 | `cidr`、`switch`、`security_group` |
| 目标值 | CIDR/IP、交换机 ID 或安全组 ID |

## 轻量云专属安全组

轻量云用户不自行创建或切换 VPC。管理员在用户配置中选择专用 NAT VPC，并在分配 VM 时由后端自动完成绑定。

每台轻量云 VM 会创建一个 VM 专属安全组：

- 安全组绑定到单台 VM，用户只能管理该 VM 的安全组规则。
- 用户不能创建全局安全组，也不能删除或切换安全组。
- 用户添加规则时仅支持 CIDR 目标，避免误引用其他用户或管理员网络资源。
- 创建端口转发时仍会自动补入该 VM 专属安全组的入站允许规则。

## 虚拟机绑定

创建、克隆、导入虚拟机请求新增：

```json
{
  "switch_id": 1,
  "security_group_id": 1
}
```

创建或更新交换机时，带宽字段使用：

```json
{
  "bandwidth_down_mbps": 100,
  "bandwidth_up_mbps": 20
}
```

旧版 `bandwidth_mbps` 字段仍兼容；如果只传旧字段，会同时作为上下行总带宽。

普通用户没有交换机、没有安全组、未选择交换机/安全组，或交换机上下行流量/带宽配额不足时，后端会拒绝创建。

管理员在「新建虚拟机」表单中只会看到自己所属的交换机和安全组，避免误把新 VM 绑定到其他用户的 VPC 资源。管理员如需查看或维护其他用户的交换机、安全组，仍在左侧「网络」页面通过用户筛选进行操作。

已存在虚拟机可在虚拟机详情页「网络管理」里的「VPC / 安全组」标签页查看当前绑定，并可随时切换安全组。切换安全组只重建 ACL，不修改网卡。

「VPC / 安全组」标签页同时提供 VPC 静态 IP 绑定。VM 已绑定 VPC 时，静态 IP 会写入该交换机独立的 dnsmasq `dhcp-hosts` 文件，IP 留空时从当前交换机 DHCP 地址池自动分配；手动输入最后一位数字时会按交换机网关所在网段补全。绑定或解绑后，运行中的 VM 仍可能需要在系统内重新获取 DHCP 或重启后才显示新地址。

旧 VM 不会强制迁移到 VPC。如果需要把旧 VM 纳入 VPC，建议先关机再切换网络，避免运行中改网卡造成断网。

运行中的 VM 保存 VPC 绑定后，宿主机侧会立即更新 OVS VLAN tag 和 ACL，但虚拟机内部已经持有的 DHCP 地址不会被宿主机主动释放。因此 `linuxtest` 这类 Linux 虚拟机可能继续显示旧 IP，直到在虚拟机内重新获取 DHCP、重启网卡，或重启虚拟机后才会拿到新交换机子网的 IP。

VPC 绑定会同步写入 VM 的 libvirt 持久 XML VLAN 配置，并在服务启动时重刷所有已绑定 VM 的运行态 OVS 端口 tag。这样 VM 重启、宿主机重启或 vnet 端口重建后仍会回到绑定的 VPC 交换机，不会重新落回默认 OVS 网段。

## 端口转发

添加端口转发后，后端会检查该 VM 是否已绑定 VPC 安全组。若已绑定，系统会自动给当前安全组补一条对应协议和内部端口的入站允许规则，来源为 `0.0.0.0/0`，并重新应用 ACL。

VPC VM 的端口转发只保留 DNAT 规则，不再写入绕过安全组的 `iptables FORWARD ACCEPT`。转发流量进入 VM 前会按目标 VM IP 命中 VPC ACL；删除对应安全组入站规则后，新的转发连接会被拒绝，已建立的 DNAT 连接也会在下一次包经过 ACL 时被拒绝。

## API

用户与管理员共用 `/api/vpc/*`，管理员可通过 `username` 参数筛选或管理指定用户：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/vpc/quota` | 查看用户 VPC 流量和带宽配额分配 |
| GET/POST | `/api/vpc/switches` | 查询或创建交换机 |
| PUT/DELETE | `/api/vpc/switches/:id` | 更新或删除交换机 |
| POST | `/api/vpc/switches/:id/traffic/reset` | 管理员重置交换机本月流量计数器 |
| GET/POST | `/api/vpc/security-groups` | 查询或创建安全组 |
| PUT | `/api/vpc/security-groups/:id` | 更新安全组名称和备注 |
| DELETE | `/api/vpc/security-groups/:id` | 删除安全组 |
| POST | `/api/vpc/security-groups/:id/rules` | 添加安全组规则 |
| DELETE | `/api/vpc/security-groups/rules/:id` | 删除安全组规则 |
| GET | `/api/vpc/acl/preview` | 管理员预览 nft ACL |
| POST | `/api/vpc/acl/apply` | 管理员应用 nft ACL，需二次验证 |
| GET | `/api/vm/:name/vpc` | 查看 VM 的 VPC 绑定 |
| PUT | `/api/vm/:name/vpc` | 更新 VM 的交换机和安全组绑定 |
| PUT | `/api/vm/:name/security-group` | 仅切换 VM 安全组 |

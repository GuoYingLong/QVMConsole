# 公网 IP 与浮动 IP 使用文档

## 功能概述

管理员可在左侧「公网 IP」页面维护公网 IP 池，并把公网 IP 绑定到指定用户的虚拟机。当前支持三种模式：

| 模式 | 说明 | VM 内是否配置公网 IP |
|------|------|----------------------|
| 1:1 NAT | 宿主机使用 DNAT/SNAT 把公网 IP 与 VM 私网 IP 做一对一映射 | 否 |
| 经典网络-路由 | 上游把公网 IP 或公网段路由到宿主机，VM 内手动配置公网 IP | 是 |
| 经典网络-桥接 | VM 接入桥接直通网桥，由上游二层网络承载公网 IP | 是 |

默认推荐使用 `1:1 NAT`。该模式兼容性最高，VM 仍保留现有 VPC/OVS 私网 IP，公网入站和出站源地址由宿主机规则完成映射。

## 公网 IP 池

新增公网 IP 时填写：

| 字段 | 说明 |
|------|------|
| 公网 IP | 要分配给 VM 的公网 IPv4 地址 |
| CIDR/掩码 | 可填写 `203.0.113.10/32` 或公网段，例如 `203.0.113.8/29` |
| 网关 | 经典网络下提供给 VM 内部配置使用 |
| 出口网卡 | NAT 模式下用于 SNAT 出口；留空时后端自动检测默认出口 |
| 支持模式 | 可限制该 IP 允许 NAT、经典网络-路由或经典网络-桥接 |
| 状态 | 空闲或保留 |

如果 `1:1 NAT` 的公网 IP 需要宿主机响应 ARP，系统会按公网 IP 和 CIDR 前缀把地址加到出口网卡上。解绑或重载规则时，系统会先清理面板托管的宿主机公网地址，再按当前绑定重新添加，避免解绑后公网 IP 仍由宿主机响应。

## 绑定与迁移

绑定公网 IP 时选择用户、VM 和模式：

- `1:1 NAT`：需要 VM 私网 IP。留空时后端会优先使用 VPC/OVS 静态绑定或 DHCP 租约，必要时自动绑定静态 IP。
- `经典网络-路由`：页面会生成 VM 内 IP、掩码和网关提示。上游必须把公网 IP 或公网段路由到宿主机。
- `经典网络-桥接`：页面会生成 VM 内配置提示。上游交换机必须允许 VM MAC 使用该公网 IP。

浮动 IP 迁移会把已有公网 IP 从旧 VM 切换到新 VM：

- NAT 模式迁移会替换 DNAT/SNAT 目标。
- 经典网络迁移会更新绑定关系和防伪造 OpenFlow 规则。
- 迁移后系统会重新应用公网 IP 规则。

绑定、解绑、迁移和重载规则都属于高风险操作，需要二次验证，并通过任务队列执行。

绑定成功后，VM 详情页的「网络与连接」卡片会显示该 VM 当前绑定的公网 IP 和绑定模式。

## 入站访问控制

`1:1 NAT` 模式不会绕过现有 VPC 安全组：

- 如果 VM IP 属于 VPC 管理网段，入站流量仍需通过安全组放行。系统会把 VM 主 IP 和公网 NAT 绑定的目标私网 IP 一并纳入同一安全组规则。
- 如果 VM IP 不属于 VPC 管理网段，系统会补充基础 FORWARD 放行规则。

经典网络模式会为运行中的 VM 端口写入 OVS OpenFlow 防伪造规则，只允许该端口使用已绑定的公网 IP 作为 IPv4 源地址和 ARP 源协议地址。若 VM 未运行或未找到 OVS ofport，规则脚本会保留注释提示，VM 启动后可在页面点击「重载规则」重新应用。

## 持久化与恢复

系统会生成：

```text
/etc/kvm-console/public-ip/rules.sh
```

每次应用前会备份旧规则到：

```text
/etc/kvm-console/public-ip/backups/
```

服务启动时会根据数据库中的公网 IP 绑定关系重新生成并应用规则，避免宿主机重启后公网 IP 映射丢失。

## 前置条件

`1:1 NAT`：

- 宿主机能使用该公网 IP。
- 上游允许该 IP 路由到宿主机，或允许宿主机出口网卡响应该 IP。

`经典网络-路由`：

- 上游把公网 IP 或公网段路由到宿主机。
- VM 内需要手动配置公网 IP、掩码和网关。

`经典网络-桥接`：

- 已创建桥接直通网桥。
- 上游交换机允许多 MAC 或允许 VM MAC 使用该公网 IP。
- VM 内需要手动配置公网 IP、掩码和网关。

## 故障排查

1. 公网入站不通：先在页面预览规则，确认 DNAT/SNAT 已生成，再检查安全组是否放行。
2. 出站源 IP 不正确：确认 SNAT 规则在通用 MASQUERADE 之前。
3. 经典网络 VM 无法联网：确认上游是否支持对应的路由或桥接条件。
4. VM 启动后经典网络防伪造未生效：点击「重载规则」，让系统重新识别 VM 的 OVS ofport。
5. 解绑后仍能 ping 通公网 IP：确认已经执行「重载规则」或解绑任务已完成；若仍通，检查宿主机出口网卡是否残留该公网地址。

# FnOS 模板克隆系统分区自动扩容

## 功能说明

FnOS 模板从模板克隆时，如果用户填写的系统盘容量大于模板原始容量，后端会在虚拟机首次启动前自动扩展系统分区。

当前流程在宿主机通过 `guestfish` 离线完成：

1. 创建基于模板的 qcow2 链式克隆磁盘。
2. 检测克隆磁盘分区表、扇区大小、分区列表和文件系统类型。
3. 选择最大的 `ext2` / `ext3` / `ext4` 分区作为 FnOS 系统分区。
4. 仅当该系统分区是最后一个分区且后方有空闲空间时，扩展分区边界到磁盘末尾。
5. 执行 `e2fsck-f` 和 `resize2fs` 扩展 ext 文件系统。
6. 继续离线注入 FnOS 首次管理员账号、密码、主机名和初始化标记。

该流程不依赖 SSH，也不会等待 FnOS 开机后再执行扩容。

## 适用布局

已在测试机模板 `FnOS.qcow2` 上确认的布局：

| 分区 | 类型 |
|------|------|
| `sda1` | EFI / vfat |
| `sda2` | FnOS 系统分区 / ext4 |

该布局下，`sda2` 位于磁盘末尾，可以安全扩展到用户创建时填写的系统盘容量。

## 限制与故障排查

- 如果系统分区后还有其他分区，后端会终止克隆并提示无法安全自动扩容，避免移动未知数据分区。
- 如果模板使用非 ext 文件系统，后端会跳过离线扩容；需要后续根据该文件系统补充专用处理逻辑。
- 如果扩容超时或 `resize2fs` 失败，请确认模板已经正常关机，且 ext 文件系统没有错误。
- FnOS 首次登录信息仍由原有离线注入流程处理，不需要模板内开启 SSH。

## 依赖

本功能复用既有宿主机依赖 `libguestfs-tools` 中的 `guestfish`，没有新增 apt 或第三方依赖。

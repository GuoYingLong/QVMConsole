# 快照配额说明

## 功能概述

本次更新为虚拟机快照增加了数量配额，区分弹性云和轻量云两种模式：

- 弹性云：按用户维度统计快照总数，默认上限为 `5`
- 轻量云：按单台 VM 统计快照数量，默认上限为 `2`

`0` 表示不限。

## 弹性云规则

- 统计范围：当前用户名下所有 VM 的快照总数
- 默认值：新建普通弹性云用户时，快照配额默认为 `5`
- 管理入口：`用户管理 -> 配置 -> 快照数量`
- 生效时机：用户创建快照前立即校验

当弹性云用户超出快照配额时，后端会拒绝创建并返回中文提示。

## 轻量云规则

- 统计范围：单台轻量云 VM 自己的快照数量
- 默认值：
  - 新登记轻量云 VM 默认 `2`
  - 轻量云已开通 VM 若自动补建单 VM 配额，也默认 `2`
- 管理入口：
  - `用户管理 -> 注册VM -> 编辑 -> 快照上限`
  - `用户管理 -> 分配VM` 时，轻量云单 VM 配额里也可直接设置
- 生效时机：该 VM 创建快照前立即校验

当轻量云 VM 超出快照配额时，只影响当前 VM，不影响同用户下其他轻量云 VM。

## 页面展示

- 用户管理列表会展示弹性云用户的 `已用快照 / 快照上限`
- VM 快照管理弹窗会显示当前 VM 的快照配额占用情况
- 当当前 VM 已达到快照上限时，页面会禁用“创建快照”按钮
- 页面创建快照时只需要填写描述，快照名称由系统自动生成。
- API 调用方也可以显式传入快照名称；名称只能使用英文字母、数字、下划线、点和短横线，且必须以英文字母、数字或下划线开头，最长 `64` 个字符。该限制用于兼容 libvirt/QEMU 内部快照任务 ID。

## 升级兼容

- 旧弹性云普通用户在数据库迁移后会自动补为默认快照配额 `5`
- 旧轻量云单 VM 配额和待开通注册记录在迁移后会自动补为默认快照配额 `2`
- 管理员账号默认不受快照配额限制

## UEFI 内存快照兼容

UEFI pflash 虚拟机创建包含内存状态的内部快照时，NVRAM 需要使用 `qcow2` 格式。新建、导入和克隆 VM 会自动使用兼容格式；老 VM 若仍是 `raw` NVRAM，运行中创建内存快照会先弹出二次确认，确认后由任务队列正常关机、转换、重新开机并继续创建快照。详细说明见 [vm-snapshot-nvram-qcow2.md](./vm-snapshot-nvram-qcow2.md)。

运行中虚拟机如果挂载了 9p/VirtFS 共享目录，libvirt 会禁用创建包含内存状态的内部快照。此时请先在虚拟机内卸载共享目录并关机移除共享挂载，或取消勾选内存状态创建仅磁盘快照。

## 运行中内存快照创建方式

运行中的虚拟机勾选“创建快照时保存虚拟机内存状态”后，页面会提供两种创建方式：

- 暂停后创建（推荐）：后端先执行 `virsh suspend`，创建内部快照后再执行 `virsh resume`。该方式一致性更稳，也更容易得到可恢复的内部快照，但快照写入期间虚拟机会暂停。
- 直接创建（实验）：后端不主动执行 `virsh suspend`，直接调用 libvirt 创建内存快照。该方式可以用来观察宿主机当前 libvirt/QEMU 的行为，但仍可能被底层短暂停顿、失败，或因为运行时写入导致一致性不如推荐方式。

API 调用方可在创建快照请求中传入 `pause_for_memory_snapshot`。字段未传时默认为 `true`，保持旧版本的暂停后创建行为；只有在 `include_memory=true` 且虚拟机处于运行中时该字段才有实际意义。

# 虚拟机 QEMU Guest Agent

## 功能说明

虚拟机高级设置中新增 `QEMU Guest Agent` 配置入口，用于控制是否在虚拟机定义中启用标准的 guest agent 通道。

QEMU Guest Agent 安装在来宾系统内部，常见用途包括：

- 让宿主机读取虚拟机内部基础信息
- 提高来宾关机、重启等协作操作的可靠性
- 为冻结文件系统后再做快照等场景提供通信能力

## 默认行为

- 默认关闭
- 启用后会在 libvirt domain XML 中写入：
  - `target name='org.qemu.guest_agent.0'`
  - `target type='virtio'`

如果虚拟机定义中原本没有 `virtio-serial` 控制器，libvirt 会在 `define` 时自动补齐所需控制器与地址，因此面板侧不需要手动写死 PCI 地址。

## 面板当前暴露字段

当前高级设置弹窗仅保留一个开关：

- `使用 QEMU Guest Agent`

以下 PVE 表单中的字段，当前面板暂不单独暴露：

- `在磁盘移动或虚拟机迁移后运行 guest-trim`
- `在备份时冻结/解冻来宾文件系统`
- `Type`

原因是当前面板主要基于 libvirt domain XML 管理虚拟机，以上字段要么属于特定备份链路策略，要么在当前实现里没有实际消费方，先不加入界面可减少误配和后续兼容负担。

## 使用建议

### Linux / Windows 服务器

建议启用，但需要虚拟机内部已安装并启动 `qemu-guest-agent` 服务，否则宿主机虽然能看到通道配置，但无法真正获取来宾协作能力。

### 普通测试机

如果没有依赖来宾信息读取、可靠关机或文件系统冻结能力，可以保持默认关闭。

## 生效规则

- 新建虚拟机：创建定义时直接写入
- 模板克隆：克隆生成定义时直接写入
- 磁盘导入：导入生成定义时直接写入
- 编辑已有虚拟机：更新持久 XML 配置

对于已在运行中的虚拟机，建议关机后修改并重新开机，以确保来宾侧能在下一次启动时正确挂载该通道。

## 兼容说明

- 旧虚拟机如果没有显式配置 guest agent 通道，详情接口会按 `未启用` 回显
- 关闭后会从 domain XML 中移除对应的 `org.qemu.guest_agent.0` 通道定义
- 若历史定义中保留了 `virtio-serial` 控制器但没有 guest agent 通道，面板仍按 `未启用` 处理

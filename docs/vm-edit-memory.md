# 虚拟机编辑内存单位调整说明

## 概述
为了统一前端交互体验和后端数据处理逻辑，虚拟机编辑页面的内存单位已从 **MB（兆字节）** 调整为 **GB（吉字节）**，与虚拟机创建时的内存单位保持一致。

## 涉及改动

### 前端调整
1. **组件路径**: `web/src/components/VmForm.vue`
2. **逻辑变更**:
   - 表单中控制编辑内存的单位由 `MB` 修改成了 `GB`。
   - 打开编辑弹窗时，会自动将列表/详情接口中返回的 `MB` 数据除以 `1024`，并 `Math.round()` 取整得到 `GB` 放入表单预显示。
   - 输入框的步长 (`step`) 、最小值 (`min`) 、最大值 (`max`) 分别调整为基于 `GB` 进行控制（例如：最小为 1GB，最大为 64GB 或随需求拓展）。

### 后端 API 调整
1. **API 路由**: `POST /api/v1/vm/:name/edit`
2. **处理逻辑**: `server/handler/vm.go`
   - 之前由客户端传 `MB` 给 `Memory` 字段的逻辑调整为：前端统一传入 `GB`。
   - 配额检查逻辑：原来计算 `deltaMemoryGB` 的公式从 `(req.Memory / 1024) - (oldMemMB / 1024)` 变为了 `req.Memory - (oldMemMB / 1024)`，直接使用 `GB` 计算增量。
   - KVM 虚拟机配置更新：调用底层的 `service.EditVMConfig` 时，由于底层 Libvirt 配置需要使用 `MB` 或者 `KB` 计量，此时在 Handler 层将传入的 `GB` 内存值乘以 **1024** 后再下发给服务层进行后续执行。

## 注意事项
- 在详情页展示的虚拟机信息（如 `web/src/views/vm/detail.vue`）依然保持原始数据（如 API 返回 MB）。为了更好地用户体验，该页面展示形式已兼容处理：大于等于 1024MB 时以 GB 展示，小于时保留 MB。
- 对于通过脚本调用的 API 使用方，请确保更新调用时的 `memory` 字段为 `GB`。

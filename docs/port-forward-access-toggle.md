# 端口转发开关与开通说明

## 功能概述

本次更新为用户配额新增了 **端口转发功能开关**，并调整了虚拟机详情页「网络管理」中的端口转发子标签页交互：

- 用户未开通端口转发时，不显示「端口转发」子标签页
- 用户已开通端口转发时，首次进入每台 VM 的端口转发页会先看到说明卡片
- 点击「立即开通」后，系统会先把当前 VM 的可用 IP 固定为静态绑定，再展示端口转发列表与操作区

## 配额规则

- 新增配额字段：`enable_port_forward`
- 默认值：`true`
- 管理员不受该开关限制
- 现有 `max_port_forwards` 数量配额继续保留，用于限制已开通用户可创建的端口转发条数

## 页面变化

### 用户管理

- 新增用户弹窗中的「端口转发」改为开关
- 编辑配额弹窗中的「端口转发」改为开关
- 只有开关开启时，才显示「转发数量」输入项
- 用户列表中的「端口转发」列会显示：
  - 已开通 / 未开通
  - 若已开通，继续显示已用数量与上限

### 邀请注册页

- 邀请详情会显示端口转发功能是否开通
- 若已开通，会继续显示端口转发数量配额

### 用户首页

- 未开通端口转发的用户，首页不再展示端口转发配额卡片

### 虚拟机详情页

- 端口转发子标签页仅对已开通用户显示
- 说明卡片的开通状态按 **虚拟机维度** 记录，不再按用户维度共用
- 当前 VM 若已经存在静态 IP 绑定，会直接视为该 VM 已开通端口转发，不重复弹出说明卡片
- 首次展示时会出现说明卡片，提示：
  - 端口转发用于将外网访问流量转发到虚拟机
  - 点击「立即开通」时，系统会自动将虚拟机当前 IP 绑定为静态地址
  - 网页访问仍受建站探测与白名单策略约束

## 后端限制

- 普通用户新增或编辑端口转发时，后端会额外校验 `enable_port_forward`
- 即使前端被绕过，未开通用户也无法通过接口创建或修改端口转发

## 测试建议

- 局部测试：
  - 关闭用户端口转发开关后，确认虚拟机详情页不显示端口转发子标签页
  - 开启开关后，确认首次进入会显示说明卡片，点击「立即开通」后会生成静态 IP 绑定并展示真实内容
  - 关闭开关后直接调用新增/编辑端口转发接口，确认后端返回拒绝
- 联调测试：
  - 已开通用户继续验证端口转发数量配额是否仍然生效

# KVM Unrestricted Guest 兼容性开关

## 功能说明

系统设置页的「宿主机设置 → 虚拟化兼容性」中提供 `KVM Unrestricted Guest` 开关，仅管理员可见。

该开关对应宿主机 Intel KVM 模块参数：

```bash
/sys/module/kvm_intel/parameters/unrestricted_guest
```

它是宿主机级参数，会影响当前宿主机上的所有 Intel KVM 虚拟机，并不是单台虚拟机独立配置。

## 适用场景

默认建议保持启用。

当宿主机运行在 VMware / ESXi 嵌套虚拟化环境中，虚拟机启动时报出类似错误时，可尝试禁用：

```text
KVM: entry failed, hardware error 0x7
```

这类错误通常会让虚拟机进入 `paused (internal-error)` 状态，继续运行时可能提示需要重置虚拟机。禁用 `unrestricted_guest` 可作为兼容性绕过方案。

## 面板位置

进入「系统设置 → 宿主机设置」后，在「虚拟化兼容性」分组中查看和切换该参数。

页面会显示：

- 当前运行时状态
- 持久配置状态
- 是否需要重载 KVM 模块或重启宿主机

## 生效方式

切换开关并完成高风险验证后，后端会写入：

```text
/etc/modprobe.d/kvm-intel-unrestricted-guest.conf
```

### 自动热重载

如果同时满足以下条件，后端会尝试重载 `kvm_intel` 模块让配置立即生效：

1. 当前没有通过 libvirt 管理的运行或暂停虚拟机
2. `kvm_intel` 模块的引用计数为 0（模块未被内核组件持有）

### 无法热重载的场景

如果以下情况之一，系统只会保存持久配置，并提示需要重启：

- 当前有虚拟机正在运行或暂停（需先关停所有虚拟机后重载模块）
- 模块引用计数不为 0，但无运行中的虚拟机（例如系统运行在 **KVM 嵌套虚拟化环境**中，内核的 KVM guest 驱动持有了模块引用）
- `modprobe -r kvm_intel` 执行失败（模块被其他进程或内核组件占用）

以上情况需要**重启宿主机**后配置才会完全生效。

## 注意事项

- 该功能需要管理员权限。
- 修改时会触发高风险验证。
- 非 Intel KVM 环境或未加载 `kvm_intel` 时，面板会显示不支持。
- 禁用后可能略微影响虚拟机早期启动阶段性能，普通物理宿主机通常不需要关闭。

## 排障建议

如果虚拟机已经处于 `paused (internal-error)`，仅保存该开关通常不能直接恢复当前坏掉的运行态。建议先强制断电该虚拟机，再重新开机验证。

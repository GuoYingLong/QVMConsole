# VM 归属映射校验

## 问题背景

面板通过 `VMAccessDir`（默认 `/etc/libvirt/vm-access`）维护“用户 -> 可管理虚拟机”的映射。

如果历史上存在已经从数据库删除、但访问映射文件仍残留的用户名，系统在按虚拟机名称反查归属时，可能会误命中这个失效用户名，进而在后续用户配额、VPC 默认资源解析等流程里出现：

- `用户不存在`
- `用户不存在: record not found`

这类报错在“新建完成后的收尾流程”中最容易出现，例如创建后自动补网络归属、按虚拟机名称推断归属用户等场景。

## 本次修复

本次修复做了两层保护：

1. `FindVMOwner` 在扫描 `vm-access` 文件时，会忽略数据库中已经不存在的用户名，避免把虚拟机误归属到失效账号。
2. `AssignVMsToUserWithQuotas` 会先校验目标用户是否存在，再写入 `vm-access` 文件，避免再次制造脏映射。

同时，重新生成 polkit 规则时也会跳过无效用户名，避免历史残留文件继续参与权限映射。

## 排障建议

如果线上或测试机已经出现过这类问题，建议检查：

- 数据库 `users` 表中的用户是否仍然存在
- `/etc/libvirt/vm-access/` 下是否残留同名旧文件

若确认是历史残留，可在确认没有业务依赖后清理失效用户名对应的映射文件。

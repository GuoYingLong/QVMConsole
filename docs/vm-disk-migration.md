# 本机虚拟机硬盘迁移

## 功能概述

管理员可以在虚拟机列表点击「迁移」，选择「迁移硬盘」，将当前宿主机上一台 VM 的指定硬盘迁移到已配置的本机存储池。

硬盘迁移不会跨节点，也不会接管用户、网络或端口转发；它只修改该 VM 指定硬盘的落盘位置。

## 迁移规则

- 运行中的 VM 自动按热迁移处理，关机 VM 自动按冷迁移处理。
- 热迁移使用 libvirt `virsh blockcopy --pivot --transient-job`，迁移完成后会把运行态硬盘切换到新路径，并同步持久化 XML。
- 冷迁移会复制硬盘文件到目标存储，更新持久化 XML，成功后删除源文件。
- 链式克隆硬盘只迁移当前活动 overlay，backing 文件保持原路径，不会合并为独立盘。
- 迁移成功后自动删除源硬盘文件；失败时保留源文件，热迁移失败会尽量清理本次创建的目标文件。
- 光驱、空盘、块设备磁盘不支持迁移。
- VM 存在外部快照时拒绝迁移硬盘，避免快照链不一致。
- 迁移任务等待或运行期间，VM 会显示为迁移中，并阻止电源、快照、磁盘、救援、密码重置和跨节点迁移等冲突操作。

## 页面操作

1. 进入「虚拟机」列表。
2. 点击目标 VM 的「迁移」。
3. 在弹窗顶部选择「迁移硬盘」。
4. 选择要迁移的硬盘和目标存储。
5. 点击「提交硬盘迁移」。
6. 到「任务中心」查看 `迁移硬盘` 任务进度。

## API

- `GET /api/vm/:name/disk-migration/options`：获取当前 VM 的硬盘迁移选项。
- `POST /api/vm/:name/disk/:dev/migrate`：提交硬盘迁移任务。

提交请求体：

```json
{
  "target_storage_pool_id": "sdb1"
}
```

返回任务类型为 `vm_disk_migrate`。

该提交接口属于高风险操作，操作标识为 `migrate_vm_disk`。管理员 API Key 按现有规则可跳过二次验证。

## 依赖

本功能不新增宿主机依赖，复用已有 `virsh`、`qemu-img` 和 `coreutils`。

# 用户网络流量月配额

## 功能概述

为每个用户的所有 VPC 交换机设置网络流量月配额（上行/下行独立）。用户创建交换机时从用户总配额中分配上下行月流量，交换机外网流量按月统计和重置。

## 核心机制

| 项目 | 说明 |
|------|------|
| **流量统计来源** | `vm_stats_records` 表中的 `net_rx_bytes`/`net_tx_bytes`（累积值） |
| **统计范围** | VPC 交换机绑定 VM 的本月外网流量增量 |
| **检查间隔** | 每 60 秒检查一次 |
| **自动重置** | 每月重置，恢复本月统计窗口 |
| **惩罚速率** | 旧版用户级非 VPC VM：下行超限 → 10Mbps，上行超限 → 1Mbps；VPC 交换机按超限方向强制 1Mbps |
| **方向独立** | 上行和下行配额独立计算，互不影响 |

轻量云用户不使用用户级流量配额。轻量云流量配额按单台 VM 配置和统计，超限后同样按方向独立处理，超限方向强制 `1Mbps`，未超限方向继续使用管理员设置的 VM 带宽。

### 与带宽配额的关系

- **带宽配额**：限制网络速率上限（Mbps），持续生效
- **流量配额**：限制每月传输总量（GB），超限后临时降速

两者独立。旧版非 VPC VM 的流量超限惩罚速率会覆盖正常的带宽均分速率；恢复后还原为正常的带宽配额均分。已绑定 VPC 的 VM 不再参与用户级月流量惩罚，避免后台检查把 VM 级速率重新写成 `10/1`，VPC 侧由交换机月配额和交换机总带宽负责控制。

带宽配额重分配会按同一用户下可读取网卡 MAC 的 VM 进行均分。若用户下存在已损坏、未定义网卡或无法从 `virsh domiflist` 读取 MAC 的历史 VM，系统会跳过该 VM 并记录警告，不会阻断其他 VM 的速率调整。

## 使用方法

### 1. 设置流量配额

**创建用户时**：在新增用户对话框的「流量配额」区域设置：
- **下行流量**：下行月配额（GB/月），0=不限制
- **上行流量**：上行月配额（GB/月），0=不限制

**编辑配额时**：同上。保存后系统会检查：如果用户当前已被限速但新配额大于已用量，自动恢复正常速率。

### 2. 查看流量使用

在用户管理列表中新增「下行流量」和「上行流量」两列：
- 显示 `已用 / 配额`（如 `1.50 GB / 10GB`）
- 使用率进度条
- 已限速时显示红色「已限速」标签

### 3. 管理员手动重置

在操作列中点击「重置流量」按钮：
- 仅在用户处于限速状态时可用
- 重置后清除本月流量偏移，恢复正常带宽
- 需二次确认

### 4. 自动重置

每月重置时自动执行：
- 恢复所有被限速用户的正常带宽
- 切换到新的月统计窗口

## VPC 交换机配额

普通用户创建交换机时需要为下行和上行分别设置月流量配额。输入框默认使用当前方向剩余最大值；如果用户某个方向的总流量配额为 `0`，表示该方向不限，交换机该方向也允许填 `0` 表示不限。多个交换机的配额合计不能超过用户总月配额；用户总配额有限时，交换机该方向配额必须大于 `0`。

交换机流量统计只统计外网方向。跨交换机或同交换机内网访问不扣减外网流量，但仍受安全组 ACL 控制。

交换机还可以分别设置下行总带宽和上行总带宽，作为该交换机所有 VM 的外网聚合速率上限。如果用户某个方向的总带宽配额为 `0`，表示该方向不限，交换机该方向也允许填 `0` 表示不限；用户总带宽配额有限时，所有交换机该方向带宽合计不能超过用户配额，且每个交换机该方向必须大于 `0`。VM 级带宽默认 `0` 表示不额外限制单台 VM；如果用户给某台 VM 设置了非 `0` 带宽，该 VM 会被单独收紧，但仍不能绕过交换机的上下行聚合上限。已绑定 VPC 的 VM 不参与用户带宽配额自动均分，流量超限恢复时也不会自动写入单机默认限速。

交换机本月下行或上行达到配额后，系统只会把超限方向的生效带宽强制降为 `1Mbps`，未超限方向继续按交换机总带宽策略运行。管理员可在「网络」-「交换机」中重置单个交换机的流量计数器；重置会把当前原始累计值写为偏移基线，并立即解除该交换机的强制限速。管理员将交换机月配额调大到高于当前已用量时，也会自动解除对应方向限速；若不手动处理，则下个月月统计窗口重置时恢复正常交换机总带宽。

## 轻量云单 VM 流量配额

轻量云 VM 的月流量直接从 `vm_stats_records` 的 `net_rx_bytes/net_tx_bytes` 相邻正增量计算：

- `net_rx_bytes` 作为 VM 下行流量。
- `net_tx_bytes` 作为 VM 上行流量。
- 只累计正增量，VM 重启或网卡计数器归零产生的负增量会被跳过。
- 每台 VM 独立保存月流量状态和 offset，不和用户级配额混算。

当某台 VM 某方向超过月配额时，后端只把该方向限速到 `1Mbps`。管理员调大配额后，如果当前已用低于新配额，会立即解除对应方向限速；月初自动恢复正常带宽。默认惩罚速率可通过系统设置表的 `lightweight_traffic_penalty_mbps` 调整。

轻量云的实时速率展示和限速执行都以 VM 网卡为准。运行中的轻量云 VM 会直接在宿主机 `vnet` 接口上设置下行 egress HTB 限速，并将上行 ingress 重定向到 IFB 接口后使用 HTB 整形，不依赖 VPC 交换机聚合统计，也不依赖 OVS 流表匹配某个 DHCP IP。运行态不会再叠加 `domiftune --live` 限速，`domiftune --config` 只用于持久化带宽配置，避免 `1Mbps` 惩罚限速时出现突发放行后长时间排队/卡顿。

## API 变更

### 用户管理

```
POST /api/user                     → 新增 max_traffic_down, max_traffic_up
PUT  /api/user/:username/quota     → 新增 max_traffic_down, max_traffic_up（保存后自动检查恢复）
GET  /api/user/list                → 返回流量配额和使用信息
POST /api/user/:username/traffic/reset → 管理员重置用户流量配额
POST /api/vpc/switches/:id/traffic/reset → 管理员重置 VPC 交换机流量计数器
```

### 用户自助

```
GET  /api/self/quota  → 返回 used_traffic_down, used_traffic_up, is_limited_down, is_limited_up
```

## 数据存储

| 表名 | 用途 |
|------|------|
| `users` | 新增 `max_traffic_down`, `max_traffic_up` 字段 |
| `user_traffic_dailies` | 旧版每日流量记录，兼容历史统计 |
| `vpc_switch_traffic_monthlies` | VPC 交换机月流量记录 |
| `vm_stats_records` | 已有表，提供 VM 网络流量原始数据 |

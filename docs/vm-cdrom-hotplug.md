# 虚拟机运行中新增光驱兼容修复

## 问题背景

部分使用 `q35` 机型的虚拟机在运行中点击“添加光驱”时，libvirt 会报错：

- `Operation not supported: disk bus 'sata' cannot be hotplugged`

原因是新增光驱原先固定走 `sata` 总线，而 `sata` 光驱不支持在运行中直接热添加。

## 现在的处理方式

后端已调整为按虚拟机状态选择新增光驱的总线：

- 运行中新增光驱：自动切换为 `scsi` 总线
- 关机状态新增光驱：优先沿用现有光驱总线；如果没有历史光驱，则保持原来的 `sata`
- 已存在光驱的“插入 / 更换 ISO”：仍然优先复用原设备，不改变已有光驱总线

当运行中的虚拟机还没有 SCSI 控制器时，后端会先补齐控制器，再执行光驱热添加。

对于 `q35` / `PCIe` 拓扑的虚拟机，还会优先根据运行中虚拟机的实际 `PCI` 拓扑信息，为 `virtio-scsi` 控制器选择空闲的 `pcie-root-port` 下游总线；如果虚拟机启动前没有预留足够的热插槽，则会明确提示先关机后再添加。

## 使用说明

在编辑虚拟机的“磁盘与光驱”页：

1. 如果已经有空光驱，优先使用“插入”
2. 如果需要额外新增一个光驱，可直接点击“添加光驱”
3. 若宿主机当前 libvirt / QEMU 环境仍不支持该热添加流程，接口会返回中文提示，建议先关机后再添加
4. 若提示 `pcie-root-port` 热插槽不足，说明这台 `q35` 虚拟机启动前没有预留足够的 PCIe 热插口，需要关机后补充控制器配置

## 影响范围

- 后端：`POST /api/vm/:name/cdrom`
- 前端：虚拟机编辑弹窗的“磁盘与光驱”页

## 兼容性说明

- 本次修改不影响创建虚拟机时一次挂载多个 ISO 的逻辑
- 不改动已有光驱的总线类型，避免对已上线虚拟机产生额外兼容风险

# 虚拟机 RTC 时间基准

## 功能说明

虚拟机高级设置中新增了 `RTC 时间基准` 选项，用于控制虚拟机硬件时钟 RTC 使用 `UTC` 还是 `本地时间`。

该选项适用于：

- 普通 ISO 创建
- 从模板克隆
- 从磁盘导入
- 编辑已有虚拟机

## 默认行为

- Windows：默认使用 `本地时间（localtime）`
- Linux / FnOS / Other：默认使用 `UTC`
- RTC 开始日期默认使用 `now`

面板会根据当前选择的系统类型自动带出推荐值：

- ISO 创建时，随系统类型和 ISO 识别结果联动
- 模板克隆时，随模板类型联动
- 磁盘导入时，随初始化类型联动

## 使用建议

### Windows

Windows 客户机通常默认按本地时间处理硬件时钟，因此建议保持 `本地时间`，可以减少来宾系统内时间偏移问题。

### Linux

Linux 系统一般建议使用 `UTC`，由系统时区层负责显示本地时间，这样更符合常见服务器和发行版默认行为。

## 生效规则

- 新建虚拟机：在生成 libvirt XML 时直接写入
- 编辑虚拟机：写入虚拟机持久配置
- 如果虚拟机已经在运行中，修改后通常需要重启虚拟机才能看到效果

## RTC 开始日期

面板新增 `RTC 开始日期` 字段：

- 默认值：`now`
- `now` 表示每次启动时使用当前时间初始化 RTC
- 如果填写固定时间，面板会将其转换为 libvirt 可识别的固定起始时间模式

支持输入格式：

- `now`
- Unix 时间戳
- `RFC3339`
- `YYYY-MM-DD HH:mm:ss`

当使用固定时间时：

- libvirt 会改为固定起始时间模式
- 此时不再按 `utc` / `localtime` 方式在启动时重新取当前时钟
- 更适合测试或排查时间相关问题，不建议用于常规业务虚拟机

## 接口与字段

前后端统一使用字段：

- `rtc_offset`
- `rtc_startdate`

可选值：

- `utc`
- `localtime`

`rtc_startdate` 默认值：

- `now`

## 兼容说明

旧虚拟机如果没有显式配置这些字段，详情接口会按 libvirt XML 当前值回显：

- 若未显式声明 RTC 时间基准，则按 `UTC` 处理
- 若未显式声明 RTC 开始日期，则按 `now` 处理

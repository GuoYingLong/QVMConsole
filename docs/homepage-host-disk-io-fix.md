# 管理员首页宿主机磁盘 IO 修复说明

## 问题现象

管理员首页中的“磁盘 I/O (KB/s)”图表在部分宿主机环境中始终显示为 0，即使宿主机本身存在真实的磁盘读写。

## 原因分析

旧实现直接使用 `/proc/diskstats` 中的设备名正则做统计，规则对 `sda`、`vda` 这类常见顶层磁盘名称匹配不完整，导致宿主机实际磁盘被漏算，最终首页图表读写速率始终为 0。

## 修复方案

当前实现调整为：

- 优先使用 `lsblk -dn -o NAME,TYPE` 枚举宿主机 `type=disk` 的顶层磁盘设备；
- 再从 `/proc/diskstats` 中汇总这些设备的累计读写扇区并转换为字节数；
- 如果 `lsblk` 不可用或返回为空，则退回到常见顶层设备名识别作为兼容兜底。

这样可以同时兼容常见的 `sd*`、`vd*`、`xvd*`、`nvme*` 等宿主机磁盘命名场景。

## 使用说明

- 修复完成后，管理员首页实时监控会在新的采样周期内展示正常的宿主机磁盘 I/O 曲线；
- 历史曲线只会对修复后的新采样数据生效，旧的错误采样记录仍可能保持为 0；
- 若宿主机 `lsblk` 命令异常，可先检查系统是否完整安装 `util-linux`。

# 虚拟机定时任务

## 功能概述

在虚拟机详情页新增「定时任务」标签页，可为单台虚拟机配置以下计划任务：

- 电源事件
  - 定时开机
  - 定时关机
- 虚拟机事件
  - 删除此虚拟机

所有定时任务都会在后台扫描后提交到任务队列执行，因此可以在「任务中心」查看执行进度；同时也会写入「调度事件」中心，便于排查自动化动作是否成功。

## 支持的计划类型

### 电源事件

- 一次性
- 每天
- 每周

### 删除虚拟机

- 仅支持一次性

原因：

- 删除属于高风险动作，误配置代价高
- 删除执行成功后会自动清理该虚拟机的全部定时任务

## 交互说明

入口：

- `虚拟机详情 -> 定时任务`

页面能力：

- 新增定时任务
- 编辑已有任务
- 启用 / 停用任务
- 删除任务
- 查看下次执行时间、最近执行时间、最近执行结果、关联任务 ID

时间说明：

- 创建和编辑时会记录当前浏览器时区
- 一次性任务按选定绝对时间执行
- 每天 / 每周任务按“时区 + 时刻”计算下一次执行时间

## 安全说明

删除虚拟机定时任务在以下场景会触发高风险验证：

- 新增并启用删除任务
- 编辑后仍为启用状态的删除任务
- 启用一个原本停用的删除任务

删除虚拟机任务触发后会走任务队列异步执行，执行完成时会额外清理：

- 虚拟机凭据
- 资源历史记录
- 连续运行记录
- VPC 绑定
- 虚拟机关联的定时任务

## 后端接口

### 获取列表

`GET /api/vm/:name/schedules`

### 创建任务

`POST /api/vm/:name/schedules`

请求体示例：

```json
{
  "event_type": "power",
  "action": "shutdown",
  "schedule_type": "weekly",
  "time_of_day": "23:30",
  "weekdays": [1, 2, 3, 4, 5],
  "timezone": "Asia/Hong_Kong",
  "enabled": true
}
```

一次性删除任务示例：

```json
{
  "event_type": "vm",
  "action": "delete",
  "schedule_type": "once",
  "run_at": "2026-05-01T18:30:00+08:00",
  "timezone": "Asia/Hong_Kong",
  "enabled": true
}
```

### 更新任务

`PUT /api/vm/:name/schedules/:id`

### 删除任务

`DELETE /api/vm/:name/schedules/:id`

## 任务中心与调度中心

### 任务中心

定时任务实际执行时会生成类型为 `vm_schedule_action` 的任务记录。

### 调度中心

调度器键值：

- `vm_scheduled_task`

调度事件中的 `vm_backend` 字段含义：

- `power`：电源事件
- `vm`：虚拟机事件

# 虚拟机引导顺序管理

## 功能概述

在编辑虚拟机的「启动与安全」标签页中，提供类似 Cockpit 风格的引导顺序管理界面。用户可以直观地查看所有可引导设备，并通过复选框和上下箭头调整引导优先级。

## 界面说明

### 编辑模式（Cockpit 风格）

打开虚拟机编辑对话框 → 「启动与安全」标签页 → 「更改引导顺序」区域：

- **设备列表**：显示虚拟机的所有可引导设备，包括：
  - 磁盘（disk）：显示文件路径、设备名、总线类型
  - 光驱（cdrom）：显示 ISO 文件路径、总线类型
  - 网络（network）：显示 MAC 地址

- **复选框**：勾选/取消勾选设备来启用/禁用从该设备引导
- **上下箭头**：调整设备的引导优先级顺序
- **设备信息**：每个设备显示类型图标、文件路径、设备标签、总线类型

### 创建模式

创建虚拟机时仍使用原有的标签式引导顺序选择（hd / cdrom / network），因为此时还没有具体的设备信息。

## 后端 API

### GET /api/v1/vm/:name

返回的 `VmDetail` 新增以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `boot_order` | `[]string` | OS 级别引导顺序，如 `["cdrom", "hd"]` |
| `boot_devices` | `[]BootDevice` | 所有可引导设备列表 |

**BootDevice 结构**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `string` | 设备类型：disk / cdrom / network |
| `device` | `string` | 设备标识：vda, sda 等 |
| `file` | `string` | 文件路径或 MAC 地址 |
| `bus` | `string` | 总线类型：virtio, sata, ide, scsi |
| `enabled` | `bool` | 是否在引导列表中启用 |
| `order` | `int` | 引导顺序（0 表示未设置） |

### PUT /api/v1/vm/:name

编辑虚拟机时传递 `boot_order` 字段（如 `["cdrom", "hd"]`），后端通过 `virt-xml` 或直接编辑 XML 来设置引导顺序。

## 修复内容

### BUG 修复
- 修复了编辑虚拟机时引导顺序总是被重置为 `['hd']` 的问题
- 现在从后端获取虚拟机的实际引导顺序和设备列表
- 引导顺序修改后能正确保存并在 Cockpit 中同步可见

### 界面改进
- 编辑模式下的引导顺序由标签式改为设备列表式（Cockpit 风格）
- 每个设备显示详细信息（文件路径、设备名、总线类型）
- 支持通过复选框启用/禁用设备引导
- 支持上下箭头调整引导优先级

## 注意事项

- 修改引导顺序在虚拟机**运行中**时设置的是配置文件，需**关机后重启**才能生效
- 引导顺序使用 libvirt 的 OS 级别 `<boot dev='...'/>` 方式，与 Cockpit 兼容

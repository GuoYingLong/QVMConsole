# 宿主机存储池

## 功能概述

管理员侧「存储池」用于直接管理物理机上的硬盘和分区，不再管理 libvirt storage pool。

页面通过系统命令实时读取宿主机块设备信息，显示设备路径、容量、文件系统、挂载点、使用率、型号和当前状态。数据库中仅保存以下面板配置：

- 显示名称
- 是否启用给普通用户
- 是否默认
- 自动挂载路径

硬盘容量、挂载状态和文件系统信息始终通过系统命令读取，避免与生产环境已有硬盘状态脱节。

## 页面可视化

存储池页面顶部新增了概览卡片和图表区域，帮助管理员快速了解存储整体状况。

### 概览卡片

页面顶部展示 4 张统计卡片：
- **总容量**：所有存储池（叶子节点）的容量总和
- **已用空间**：已挂载存储池的已用空间总和
- **可用空间**：已挂载存储池的可用空间总和
- **存储池数量**：物理硬盘数量

### 图表区域

**存储池容量分布（环形饼图）**
- 展示各存储池容量占比
- 鼠标悬浮可查看详细信息（已用、可用、使用率）
- 环形中心显示总容量数值

**存储池容量对比（堆叠柱状图）**
- 每个已挂载存储池显示为一根柱子
- 柱子分为"已用"和"可用"两段
- 已用段颜色根据使用率变化：< 70% 蓝色、70%~89% 橙色、>= 90% 红色

### 数据统计规则

为避免 lsblk 父子设备重复计算：
- 统计对象：递归遍历树形结构，收集所有叶子节点（无 children 或 children 为空）
- 总容量：所有叶子节点中 size > 0 的设备求和
- 已用/可用：仅统计叶子节点中 mountpoints 非空（已挂载）的设备
- 存储池数量：顶层物理盘数量（type=disk）

## 创建虚拟机时的落盘规则

- 管理员创建虚拟机时可以选择任意当前可用的挂载设备。
- 普通用户只能选择管理员已启用的设备。
- 留空时使用管理员设置的默认存储池。
- 如果没有默认存储池，系统回退到现有克隆磁盘目录 `clone_dir`。
- 本期仅覆盖普通创建和模板克隆，不覆盖编辑已有虚拟机时新增磁盘，也不覆盖导入虚拟机。

虚拟机磁盘会创建到：

```text
<挂载路径>/vm-disks
```

格式化并挂载的硬盘默认挂载到：

```text
/var/lib/kvm-storage/<设备ID>
```

## 格式化并挂载

格式化挂载是高风险操作，需要二次验证，并通过任务队列执行。

当前策略：

- 允许覆盖已有文件系统或分区信息。
- 拒绝格式化已挂载设备。
- 拒绝格式化根分区、启动分区等系统关键设备。
- 拒绝格式化只读设备、光驱、loop 设备。
- 使用 `mkfs.ext4 -F` 创建 ext4 文件系统。
- 使用 UUID 写入 `/etc/fstab`，挂载参数为 `defaults,nofail`。
- 自动创建 `vm-disks` 目录并设置 libvirt 可访问权限。
- 自动补充 libvirt / AppArmor 对 `/var/lib/kvm-storage` 的访问规则，确保自定义存储池中的普通磁盘和外部快照 overlay 都能被 `virt-aa-helper` 与 QEMU 读取。

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/storage-pool/list` | 获取宿主机硬盘列表 |
| `GET` | `/api/storage-pool/vm-targets` | 获取创建虚拟机可选存储位置 |
| `GET` | `/api/storage-pool/:id` | 获取单个硬盘详情 |
| `PUT` | `/api/storage-pool/:id/config` | 更新显示名和启用状态 |
| `POST` | `/api/storage-pool/:id/default` | 设为默认存储位置 |
| `POST` | `/api/storage-pool/:id/format-mount` | 提交格式化并挂载任务 |
| `GET` | `/api/storage-pool/all-isos` | 获取系统设置中全局 ISO 目录的 ISO 列表 |

## 依赖命令

存储池管理复用宿主机基础工具：`lsblk`、`findmnt`、`df`、`blkid`、`wipefs`、`mkfs.ext4`、`mount`。

这些命令分别来自 `util-linux`、`coreutils`、`e2fsprogs` 等常见系统包。

若宿主机启用了 AppArmor，安装脚本和后端运行时会写入本地 libvirt AppArmor 规则文件；该处理不新增 apt 依赖。

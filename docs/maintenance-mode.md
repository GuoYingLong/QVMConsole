# 维护模式

## 功能目标

系统设置新增“维护模式”，用于宿主机需要连续多次重启、升级或维护时，提前收敛运行中的虚拟机资源，避免每次重启后又自动拉起相关服务。

## 启用效果

管理员在“系统设置”中启用维护模式并完成二次验证后，系统会异步执行以下动作：

1. 关闭所有运行中的虚拟机。
2. 停用配置中的 systemd 服务，且不再随主机开机自动启动。

维护模式开启后，会立即阻止以下操作：

- 虚拟机启动
- 虚拟机重启
- 通过创建/克隆/导入/重装/救援启动等隐式触发开机的流程

## 默认服务列表

默认维护服务列表为：

```text
kvm-console.service
libvirtd.service
libvirtd.socket
libvirtd-ro.socket
libvirtd-admin.socket
```

说明：

- 该列表可在系统设置中修改，支持换行或逗号分隔。
- `kvm-console.service` 不会被维护模式禁用；即使手动加入列表，系统也会自动跳过，以确保主机重启后面板仍能自动启动。
- 为了保证当前页面还能继续操作，“启用维护模式”任务不会主动停止当前正在运行的面板进程。

## 关闭维护模式

关闭维护模式并保存后，系统会异步执行恢复任务：

1. 恢复维护服务列表中的开机自启。
2. 尝试重新启动这些服务。

注意：

- 关闭维护模式不会自动重新启动之前关闭的虚拟机。
- 面板服务不参与维护模式停用，主机重启后仍会按 systemd 配置自动启动。

## 二次验证

维护模式开关属于敏感操作：

- 启用维护模式需要二次验证。
- 关闭维护模式需要二次验证。
- 开发环境模式开启时，仍会按现有规则绕过高风险验证。

## 任务中心

维护模式操作会进入任务中心，任务类型包括：

- `enter_maintenance_mode`
- `exit_maintenance_mode`

建议在任务中心确认以下结果：

- 是否还有虚拟机未正常关闭
- 是否有服务停用或恢复失败

## 建议使用方式

1. 先通知用户维护窗口。
2. 在系统设置中启用维护模式。
3. 到任务中心确认资源收敛完成。
4. 执行宿主机重启、升级或其他维护操作。
5. 维护完成后直接登录面板，再关闭维护模式。

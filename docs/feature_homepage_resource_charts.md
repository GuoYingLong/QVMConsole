# 首页 24 小时资源监控图表使用说明

## 新增功能介绍

随着系统的使用，对资源瓶颈的排查和直观的系统状态监控变得尤为重要。
本次更新在全系统范围内重构了首页仪表盘，使得首页不仅具备精美、现代的 UI 设计，更加入了详细的图表资源可视化监控，帮助管理员和普通用户快速掌握历史资源调用情况。

### 1. 物理机资源 24 小时监控 (对管理人员及全员开放)
系统首页的顶部现在会展示整个物理机（宿主机）过去 24 小时内的资源统计图：
- **CPU 使用率**
- **内存使用率**
- **网络流量 (KB/s)**：显示包含所有可用物理网卡的下行/上行网络流量汇总。
- **磁盘 I/O (KB/s)**：显示核心物理硬盘的读取及写入流量汇总。
管理员可以由此洞悉宏观系统级别的资源开销和状态。

### 2. 虚拟机资源折叠列表
- 替代了原本简单枯燥的表格或信息流卡片，取而代之的是包含所有权属虚拟机的**折叠面板**展示。
- 每个折叠面板上展示了虚拟机的基础信息（如：状态、VCPU核数、内存、分配的 IP 等）。
- 当点击需要查看的虚拟机展开详情，系统将自动挂载图表组件并立即呈现该虚拟机过去 24 小时的**CPU、内存、网络、以及磁盘 I/O 数据**，与虚拟机详情页面的图表体验保持统一。

## 页面操作指南
1. 登录面板后即进入首页视图。
2. **概览数据查看**：顶部第一栏展示当前的即时总览配额及用量、物理机核心数/内存使用情况。
3. **物理机图表操作**：往下滚动到"物理机 24小时资源图表"区域。图表可拖拽底部滚动条进行时间窗口缩放。图表右上角的切换按钮支持从历史查询切回实时数据采集。
4. **虚拟机图表展开**：在"虚拟机 24小时资源追踪"列表，点击对应的行即可无缝展开对应的图标记录监控，无需跳转页面即可洞悉该虚拟机在过去一天内的资源状态起伏。展开下方提供了按钮，可一键前往该 VM 进行配置编辑及 VNC 控制。

## 技术性底层修改

### 数据库改动
系统执行了新表映射迁移，创建了 `host_stats_records` 表：
- 每隔 60 秒，底层服务会持续抓取系统自身的 `nproc`, `top`, `free`, 以及 `/proc/net/dev`, `/proc/diskstats` 文件信息，将增量转化为 KB 级别网络流量存储。

### 前端改动
- 提取了原来 `src/views/vm/detail.vue` 中的图表渲染代码，创建了高度可复用的子组件 `src/components/ResourceCharts.vue`。支持指定 `type: "host" | "vm"` 切换拉取的数据接口。
- `dashboard/index.vue` 面向新布局做了 Vue 模板与 Element UI Plus 组件体系结合的整面重写。

*注意：物理机的图表如果当前没有数据记录，系统将在后台服务启动一段时间（至少轮询间隔之后）才会渲染出线条。网络和 I/O 折线图的跳跃值做了去极值和平滑处理，消除异常网络重置。*

# 虚拟机导出/导入功能

## 功能概述

虚拟机导出/导入功能允许用户：
- **导出虚拟机磁盘**到"我的存储 → 虚拟磁盘"目录  
- **上传磁盘文件**（.qcow2/.raw/.vmdk/.vhd 等）到虚拟磁盘目录  
- **从磁盘文件导入**创建新虚拟机  

## 导出虚拟机

### 操作步骤

1. 在虚拟机列表页面，找到需要导出的虚拟机
2. 点击操作列的"更多"下拉菜单
3. 选择"导出虚拟机"
4. 确认导出后，系统会提交后台任务（可在任务中心查看进度）
5. 导出完成后，磁盘文件会保存在"我的存储 → 虚拟磁盘"标签页

### 导出机制

| 磁盘类型 | 导出方式 | 说明 |
|---------|---------|------|
| 链式克隆磁盘 | `qemu-img convert` 合并导出 | 合并 backing file 为独立 qcow2 文件 |
| 独立磁盘 | `cp --sparse=always` 复制 | 保持稀疏文件特性，节省空间 |

### 注意事项

- 导出前会**自动校验用户存储配额**，空间不足会被拒绝
- 链式克隆磁盘导出后文件大小 = 虚拟磁盘容量（非实际占用）
- 导出的文件命名格式：`<虚拟机名>-export-<时间戳>.qcow2`

## 导入虚拟机

### 操作步骤

1. 准备磁盘文件：
   - 从导出获得的磁盘文件，或
   - 在"我的存储 → 虚拟磁盘"上传外部磁盘文件
2. 在虚拟机列表点击"导入虚拟机"按钮
3. 填写表单：
   - **虚拟机名称**（必填）：英文字母开头
   - **磁盘文件**（必填）：从虚拟磁盘列表选择
   - **磁盘来源**（管理员可选）：支持"从我的存储选择"和"输入绝对路径"两种方式
   - **存储位置**（管理员可选）：选择磁盘最终落盘位置，对齐 ISO/模板创建方式的存储位置表单
   - **CPU / 内存**（必填）
   - **初始化类型**（可选）：
     - `不初始化`：仅创建 VM，不做任何配置
     - `Linux`：SSH 初始化（设置主机名/用户/密码等）
     - `Windows`：仅创建 VM（使用 UEFI + TPM 模板）
     - `其他`：仅创建 VM
4. 确认导入后，系统会提交后台任务

### 管理员绝对路径导入

管理员在新建虚拟机 → 导入模式中，可选择"输入绝对路径"方式：
- **磁盘路径**：输入宿主机上磁盘文件的绝对路径，如 `/data/disk.qcow2`
- **格式自动转换**：非 qcow2 格式（raw/vmdk/vhd/vhdx/img 等）将自动通过 `qemu-img convert` 转换为 qcow2
- **存储位置**：通过存储位置下拉框（与 ISO/模板创建方式一致）选择最终落盘目录
- **多磁盘导入**：管理员可通过「额外磁盘导入」添加多块磁盘，在系统盘创建完成后依次挂载
- 后端异步任务处理：检测格式 → 转换（如需要）→ 删除原文件（如不保留）→ 移入目标存储 → 创建 VM 定义 → 启动

| 参数 | 说明 |
|------|------|
| disk_path | 磁盘文件的绝对路径（主磁盘） |
| disk_file | 存储文件名（主磁盘 storage 模式） |
| disk_source_type | path/storage |
| storage_pool_id | 目标存储池 ID（为空则使用默认存储） |
| copy_disk | true=保留原磁盘文件，false=不保留（默认） |
| extra_import_disks | 额外导入磁盘列表，每项含 disk_path/disk_file/disk_source_type/storage_pool_id/copy_disk/bus |

### Linux 初始化配置

选择 Linux 初始化时，需要额外配置：

| 字段 | 说明 |
|------|------|
| 主机名 | 新虚拟机的主机名（默认=虚拟机名称） |
| 新用户名 | 要在新 VM 中创建的用户 |
| 新密码 | 新用户的密码 |
| 模板 root 密码 | 磁盘文件中原有的 root 密码（SSH 连接需要） |
| 模板用户名 | 磁盘文件中原有的普通用户名（用于删除旧用户） |

### 高级选项

| 选项 | 默认值 | 说明 |
|------|--------|------|
| 启动类型 | BIOS | 选择 BIOS 或 UEFI |
| 机器类型 | Q35 | Q35（推荐）或 PC (i440FX) |
| 网卡模型 | VirtIO | VirtIO（推荐）/ e1000e / e1000 / rtl8139 |
| 开机自启 | 关闭 | 宿主机启动时自动启动虚拟机 |

### 注意事项

- 用户可选择**不保留原文件**（推荐，不占额外空间）或**保留原磁盘文件**
- 如果存储目录和虚拟机目录跨设备，不保留操作会自动回退到复制+删除原文件
- 支持的磁盘格式：`.qcow2`、`.raw`、`.vmdk`、`.vhd`、`.vhdx`、`.img`
- 导入虚拟机与新建虚拟机**共用同一表单**，通过顶部"创建方式"切换

## 编辑虚拟机时导入磁盘

管理员在编辑虚拟机 → 磁盘管理页面，点击"挂载已有磁盘"按钮：
- **磁盘来源**：支持"从我的存储选择"和"输入绝对路径"
- **绝对路径模式**：输入宿主机路径 → 选择目标存储位置 → 非 qcow2 自动转换 → 提交异步任务
- **存储模式**：与原有挂载方式一致，直接同步操作

| 参数 | 说明 |
|------|------|
| disk_path | 磁盘文件的绝对路径 |
| storage_pool_id | 目标存储池 ID |
| copy_disk | 是否保留原文件 |
| bus | 磁盘总线类型（virtio/scsi/sata/ide） |

## 虚拟磁盘管理

在"我的存储"页面新增"虚拟磁盘"标签页：

- **上传**：支持上传 .qcow2/.raw/.vmdk/.vhd/.vhdx/.img 格式
- **列表**：查看所有导出/上传的磁盘文件
- **下载**：下载磁盘文件到本地
- **删除**：删除不需要的磁盘文件

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/self/vm/export` | 导出虚拟机 |
| POST | `/api/self/vm/import` | 导入虚拟机（从存储） |
| POST | `/api/vm/import-disk` | 管理员通过绝对路径导入磁盘创建虚拟机（需二次验证，支持多磁盘） |
| POST | `/api/vm/:name/disk/import` | 管理员为已有虚拟机通过绝对路径导入磁盘（异步任务） |
| GET  | `/api/self/storage/files/disk` | 列出虚拟磁盘文件 |
| POST | `/api/self/storage/upload/disk` | 上传磁盘文件 |
| DELETE | `/api/self/storage/file/disk/:filename` | 删除磁盘文件 |
| GET  | `/api/self/storage/download/disk/:filename` | 下载磁盘文件 |

# 虚拟机编辑引导方式

## 功能说明

在“编辑虚拟机”弹窗的“启动与安全”标签页中，新增了“引导方式”编辑项，支持在已有虚拟机上切换以下模式：

- `BIOS`
- `UEFI`
- `UEFI + 安全引导`

界面中会固定显示风险提示：

> 更改引导会导致原有已安装的操作系统无法启动

用于提醒用户在切换引导方式前确认当前系统镜像、分区表和固件兼容性。

## 使用位置

路径：

1. 打开虚拟机详情
2. 点击“编辑虚拟机”
3. 进入“启动与安全”
4. 在“引导方式”区域修改并保存

## 行为说明

- 仅当用户实际修改了引导方式时，前端才会向后端提交 `boot_type`
- 如果只是打开编辑弹窗并保存其他配置，不会覆盖当前虚拟机已有的引导方式
- 运行中或暂停中的虚拟机不允许修改引导方式，需要先关机
- 切换到 UEFI 模式时，后端会自动补齐所需的固件/NVRAM 配置
- 切换到 `UEFI + 安全引导` 时，会额外补齐安全引导相关固件配置
- 新建虚拟机、模板克隆和原生链式克隆都会在 `virt-install --print-xml` 后按 `boot_type` 再归一化一次 XML，避免宿主机默认把普通 `UEFI` 生成成 `UEFI + 安全引导`

## 接口影响

编辑虚拟机接口 `PUT /api/v1/vm/:name` 现支持以下新增字段：

```json
{
  "boot_type": "bios"
}
```

可选值：

- `bios`
- `uefi`
- `uefi-secure`

## 注意事项

- 该修改属于高风险兼容性变更，建议先确认系统原本就是以目标方式安装
- 从 BIOS 改到 UEFI，或从 UEFI 改到 BIOS，常见情况下会因为原系统启动项不匹配而无法启动
- 如果只是临时排查启动问题，建议优先通过“编辑虚拟机 XML”确认底层配置，再决定是否切换引导方式

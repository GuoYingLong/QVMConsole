# 虚拟机连续运行时间记录

## 功能说明

为每台虚拟机新增“连续运行时间”记录，用于展示该虚拟机自最近一次进入运行态以来已经持续运行了多久。

- 列表页新增“连续运行”列
- 详情页 Hero 区和基本配置卡片显示连续运行时长
- 后端通过 SSE 推送详情/列表时会一并携带连续运行时间字段

## 数据字段

虚拟机列表和详情接口新增以下字段：

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| `continuous_runtime_seconds` | `int64` | 当前连续运行秒数 |
| `continuous_running_since` | `string` | 当前这次连续运行的开始时间，格式为 `YYYY-MM-DD HH:mm:ss` |

说明：

- 当虚拟机状态为 `running` 或 `paused` 时，后端会返回连续运行时长
- 当虚拟机已关机时，以上字段会清空为 `0` / 空字符串

## 实现方式

后端在内存中维护每台虚拟机的运行状态快照，记录：

- 最近一次观察到的状态
- 当前这次连续运行的开始时间

状态同步来源：

1. 后台资源采集器每 10 秒扫描一次 `running / paused` 虚拟机
2. 列表、详情接口在读取虚拟机状态时也会顺手刷新记录
3. 面板内执行开机、关机、断电、重启、重置时，会立即更新记录

## 兼容说明

- 该功能不依赖 libvirt 数据库，仍以命令探测为主
- 对于功能上线前已经在运行中的虚拟机，首次被系统接管时会从“首次观察到的时间点”开始计时，而不是追溯历史真实开机时间
- 面板服务重启后，内存记录会清空；仍在运行的虚拟机会在重启后首次被观察到时重新开始计时
- 删除虚拟机时会同步清理其内存记录

# 新建虚拟机支持多 ISO 挂载

## 功能说明

新建虚拟机时，`ISO 镜像` 选择器现在支持一次选择多个 ISO 文件。

- 第 1 个 ISO：作为主安装盘
- 第 2 个及之后的 ISO：作为额外挂载光驱
- 额外 ISO 会在虚拟机定义阶段直接写入 XML，不依赖开机后再热插拔

这样可以覆盖一些常见场景，例如：

- Windows 安装盘 + `virtio-win.iso`
- 系统安装盘 + 驱动盘
- 系统安装盘 + 工具盘 / 固件盘

## 表单行为

- 选择多个 ISO 后，表单会按选择顺序展示挂载顺序
- 系统类型、系统版本、最小磁盘大小等自动识别逻辑仍以首个 ISO 为准
- 预览区会显示主 ISO，并额外提示还挂载了多少个 ISO

## 接口兼容

创建虚拟机接口新增兼容字段：

- `iso_paths: string[]`

兼容规则：

- 旧调用只传 `iso_path` 时，行为不变
- 新调用可同时传 `iso_path` 与 `iso_paths`
- 后端会自动去重、去空值，并以最终列表中的第 1 个 ISO 作为主安装盘

## 普通用户限制

普通用户通过 `/api/self/vm/create` 创建虚拟机时：

- `iso_paths` 中的所有路径都必须属于自己的 ISO 存储目录
- 若任意一个 ISO 不在自己的目录下，接口会直接拒绝

## 使用建议

- 如果需要系统自动识别来宾类型，请把真正的系统安装盘放在第一个
- 如果只是附加驱动或工具盘，建议放在后面，避免覆盖自动推荐结果

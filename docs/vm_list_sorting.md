# 虚拟机列表排序功能说明

## 1. 功能概述

本项目在虚拟机列表中增加了“创建时间”字段，并支持以下排序功能：
1. **默认排序**：进入虚拟机列表时，默认根据“创建时间”从最新到最旧（倒序）排列。
2. **状态排序**：支持点击“状态”列的表头，根据虚拟机当前的运行状态进行排序（运行中 > 已暂停 > 已关机）。
3. **扩展排序**：支持点击“创建时间”列的表头再次变更排序方式。

## 2. 技术实现

### 2.1 后端实现 (`server/service/libvirt.go`)
- 扩展了 `VmInfo` 和 `VmDetail` 结构体，增加 `CreatedAt string` 字段（JSON名称为 `created_at`）。
- **`ListVMs`** 获取虚拟机列表时，使用 `stat` 命令批量查询所有虚拟机配置文件 (`/etc/libvirt/qemu/*.xml`) 的内容，通过文件的元数据时间（首选 `%W` 文件创建时间，若不支持则降级使用 `%Y` 更新时间）来获取虚拟机的创建时间。
- **`GetVM`** 查询单个详细信息时也支持对单个 XML 文件的元数据提取创建时间。

采用上述方式在不依赖独立数据库的情况下直接利用操作系统层面的文件系统属性来获得创建时间，保证了和原生 KVM/libvirt 状态的一致性。

### 2.2 前端实现 (`web/src/views/vm/list.vue`)
- 在数据表格的列中增加了 `<el-table-column prop="created_at" label="创建时间" />`，并通过 `sortable` 属性支持前端排序。
- 为“状态”列增加 `sortable` 属性，并结合 `:sort-method="sortStatus"` 定制了针对虚拟机状态（Running / Paused / Shut off）权重的特定排序逻辑。
- 利用 Element Plus 表格的 `default-sort` 属性将默认排序配置为 `{ prop: 'created_at', order: 'descending' }`。

## 3. 注意事项
- 由于底层依赖文件的时间戳，如果手动更改或使用系统自带的工具迁移 `/etc/libvirt/qemu/` 中的 XML 文件，可能导致“创建时间”发生变化（特别是文件系统的 `%W` 时间未被保留时）。
- 若宿主机文件系统不支持查阅 `%W` (Create 属性) 时间，则该时间自动回退至最后一次修改 `%Y` (Modify 属性) 时间。
- 该排序目前由前端接管全量返回数据的交互排序，能提供更为迅速的用户体验。

# 用户封禁

## 功能概述

管理员现在可以在「用户管理」页面对普通用户执行封禁与解封。

- 封禁/解封属于高风险操作，管理员需要先完成二次验证。
- 封禁不会删除用户现有资产。
- 封禁会立即让该用户当前登录状态失效。
- 封禁任务会异步执行，并尽量关闭该用户下所有运行中的虚拟机。
- 封禁时会同步关闭该用户的 SSH 访问。
- 解封后用户可以重新登录，但之前被关闭的虚拟机不会自动恢复启动。

## 适用范围

- 仅支持已激活的普通用户。
- 管理员账户不允许封禁。
- 待激活邀请用户不支持封禁或解封。

## 后端行为

### 封禁

接口：

- `PUT /api/user/:username/status`

请求体：

```json
{
  "status": "disabled"
}
```

返回：

- 提交异步任务，响应中包含 `task_id`

任务执行流程：

1. 先把账户状态改为 `disabled`
2. 清空登录验证状态并使已有 token 失效
3. 关闭该用户 SSH 访问
4. 关闭该用户名下所有运行中的虚拟机

### 解封

接口：

- `PUT /api/user/:username/status`

请求体：

```json
{
  "status": "active"
}
```

返回：

- 直接同步生效

说明：

- 解封只恢复账户状态，不会自动开启 SSH，也不会自动启动之前关闭的资源。

## 前端提示

- 用户状态会显示为「正常 / 待激活 / 已封禁」
- 已封禁用户的 SSH 开关会被禁用，避免误操作
- 封禁操作会提示“关闭资源但不删除资产”的影响

# 虚拟机删除 - 磁盘管理

## 功能说明

删除虚拟机时，系统会弹出磁盘选择表单，用户可以选择哪些磁盘需要删除、哪些需要保留。

### 删除流程

1. 在虚拟机列表中，点击「更多 → 删除」
2. 系统弹出磁盘选择对话框，列出该虚拟机的所有 qcow2 磁盘
3. 所有磁盘默认勾选（表示删除）
4. **系统盘**（第一个磁盘）不可取消勾选，必须随虚拟机删除
5. 如果取消某些磁盘的勾选，系统会提示这些磁盘将转移到「我的存储 → 虚拟磁盘」中
6. 点击「确认删除」后：
   - 勾选的磁盘将被删除
   - 未勾选的磁盘将自动转移到用户存储池的虚拟磁盘目录
   - 转移前会校验存储配额是否足够

### 批量删除

选择多台虚拟机后点击「删除」按钮，将弹出批量删除确认框。批量删除模式下会直接删除所有磁盘，不支持磁盘选择。

### 存储配额校验

当用户选择保留（不勾选）某些磁盘时，系统会先检查用户是否已开通「我的存储」：
- **未开通**：显示红色提示，禁用确认按钮，要求用户先去「我的存储」页面初始化存储池
- **已开通**：进一步检查存储配额是否有足够空间。如果空间不足，将提示用户先释放存储空间

### 磁盘管理页面 - 单个磁盘操作

在虚拟机编辑页面的磁盘管理中，每个磁盘右侧的操作按钮提供两个选项：
- **删除**：卸载磁盘并删除磁盘文件
- **转移到我的存储**：卸载磁盘后将磁盘文件移动到用户的「我的存储 → 虚拟磁盘」目录。转移前同样会校验存储池初始化状态和配额。

## API 接口

### 获取 qcow2 磁盘列表

```
GET /api/vm/:name/qcow2-disks     (管理员)
GET /api/self/vm/:name/qcow2-disks (普通用户)
```

返回示例：
```json
{
  "code": 200,
  "data": [
    {
      "device": "vda",
      "path": "/var/lib/libvirt/images/clones/myvm.qcow2",
      "capacity_gb": "20.00",
      "format": "qcow2",
      "is_system": true,
      "size_bytes": 3221225472
    },
    {
      "device": "vdb",
      "path": "/var/lib/libvirt/images/clones/myvm-vdb.qcow2",
      "capacity_gb": "50.00",
      "format": "qcow2",
      "is_system": false,
      "size_bytes": 1073741824
    }
  ]
}
```

### 删除虚拟机（带磁盘选择）

```
DELETE /api/vm/:name              (管理员)
DELETE /api/self/vm/:name         (普通用户)
```

请求体（可选）：
```json
{
  "delete_disks": ["/path/to/disk1.qcow2"],
  "transfer_disks": ["/path/to/disk2.qcow2"]
}
```

- `delete_disks`: 要删除的磁盘文件路径列表
- `transfer_disks`: 要转移到用户存储的磁盘文件路径列表
- 如果不提供请求体，默认删除所有磁盘（向后兼容）

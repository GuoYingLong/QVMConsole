# 救援系统

## 功能概述

救援系统允许管理员/用户通过挂载救援 ISO 来修复无法正常启动的虚拟机。启动救援系统时，系统会自动将磁盘和网卡切换为兼容模式（SATA / e1000e），确保救援系统能够识别虚拟机的硬件设备。

## 配置

### 管理员设置

1. 进入 **系统设置** 页面
2. 在 **救援系统** 分区中，从下拉框选择一个 ISO 文件作为救援系统
3. 点击 **保存设置**

> 也可通过环境变量 `KVM_RESCUE_ISO` 配置 ISO 路径（重启后生效）

## 使用方法

### 启动救援系统

1. 在虚拟机列表中，找到目标虚拟机
2. 点击操作列的 **更多** → **启动救援系统**
3. 确认操作后，系统将自动执行以下步骤（异步任务）：
   - 强制关闭虚拟机
   - 记录原始配置（磁盘总线、网卡类型、引导顺序）
   - 将磁盘总线切换为 **SATA**
   - 将网卡切换为 **e1000e**
   - 挂载救援 ISO
   - 设置 CDROM 优先引导
   - 重新启动虚拟机（从救援 ISO 引导）
4. 在 **任务中心** 查看进度

### 关闭救援系统

1. 虚拟机处于救援模式时，**更多** 菜单会显示 **关闭救援系统**
2. 点击后确认操作，系统将自动执行：
   - 强制关闭虚拟机
   - 弹出救援 ISO
   - 恢复原始磁盘总线类型
   - 恢复原始网卡类型
   - 恢复原始引导顺序
   - 重新启动虚拟机（正常引导）
3. 在 **任务中心** 查看进度

## API 接口

### 启动/关闭救援系统

```
POST /api/vm/:name/rescue
```

| 参数 | 类型 | 说明 |
|------|------|------|
| action | string | `start` 启动救援 / `stop` 关闭救援 |

**响应示例：**
```json
{
  "code": 200,
  "message": "启动救援系统任务已提交",
  "data": { "task_id": 1 }
}
```

## 注意事项

- 启动/关闭救援系统都会**强制关闭**虚拟机，请确保重要数据已保存
- 救援模式下磁盘使用 SATA 总线、网卡使用 e1000e，关闭后会自动恢复原始配置
- 原始配置保存在 `/tmp/_rescue-{vmName}.json`，请勿手动删除
- 虚拟机列表中 `in_rescue` 字段标识当前是否处于救援模式
